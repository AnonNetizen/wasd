# MVP 设计说明

> **AI 修改说明**：修改本文档前先读 `MinimumViableProduct/README.md` 与根目录 `AGENTS.md`。
> 本文档是 MVP 玩法范围的权威说明；改玩家能力、敌人刷新、胜负目标或系统边界时，必须同步 `MinimumViableProduct/docs/开发计划.md` 与 `MinimumViableProduct/docs/MVP决策记录.md`，必要时同步根目录 ADR。

---

## 1. 一句话目标

做一个极小可玩验证：玩家站在中心不能移动，只能用方向键或手柄 D-pad / 摇杆切换四方向射击，敌人从四个方向刷出并逼近玩家，用来验证“快速实现 + 记录经验 + 迭代判断”的工作流。

## 2. 核心循环

```text
开始 → 敌人从四方向刷出 → 玩家用方向键切换射击方向 → 自动射击清怪 → 记录生存时间 / 击杀数 → 死亡或重开 → 复盘经验
```

## 3. 玩家规则

| 项 | MVP 规则 |
|----|----------|
| 位置 | 固定在场景中心，不响应 WASD 或其他移动输入 |
| 瞄准 | 方向键上 / 下 / 左 / 右，或手柄 D-pad / 摇杆切换当前射击方向 |
| 开火 | 自动按固定间隔发射子弹 |
| 初始方向 | 默认向上，或首次方向键输入后确定 |
| 受伤 | 敌人接触玩家时扣血或直接失败，具体数值后续实现时定 |

## 4. 敌人规则

| 项 | MVP 规则 |
|----|----------|
| 刷新方向 | 只允许上 / 下 / 左 / 右四个刷怪点 |
| 移动目标 | 朝玩家中心点直线推进 |
| 行为 | 不绕路、不远程攻击、不特殊技能 |
| 难度 | 初版固定刷怪间隔；之后可按时间缩短间隔 |

## 5. 子弹规则

- 子弹沿当前射击方向直线飞行。
- 子弹命中敌人后造成伤害并销毁；敌人死亡后计入击杀数。
- 初版只做单发直线子弹，不做穿透、分裂、追踪、暴击。

## 6. 最小系统清单

| 系统 | 最小职责 |
|------|----------|
| `Player` | 固定位置、记录当前射击方向、承受伤害 |
| `AimInput` | 读取方向键与手柄 InputMap action，并输出四方向向量 |
| `Weapon` | 按间隔自动生成子弹 |
| `Bullet` | 直线移动、命中敌人 |
| `Enemy` | 朝玩家移动、接触玩家 |
| `Spawner` | 在四个方向生成敌人 |
| `GameSession` | 记录生存时间、击杀数、失败 / 重开 |
| `HUD` | 显示血量、时间、击杀数 |

## 6.1 MVP 数据配置

- MVP 使用 `MinimumViableProduct/client/data/mvp_config.json` 集中维护核心可调数值。
- 当前覆盖玩家 HP、受伤反馈、手柄死区、武器 / 子弹、敌人、刷怪节奏和背景关键尺寸。
- MVP 暂不引入完整项目的 `DataLoader`、JSON Schema、热重载或词表生成常量；如果某套配置机制要迁移正式项目，需要另行复盘并升级为根目录 ADR / GDD 设计。

## 6.2 MVP 视觉方向

- 当前采用无贴图的几何霓虹占位风格，优先保证可读性和快速迭代。
- 背景用深色网格、四方向通道和中心准星强调“固定中心 + 四方向压力”。
- 玩家、敌人、子弹都用 `_draw()` 绘制，避免在 MVP 阶段引入美术资源管理成本。
- 正式美术风格是否沿用该方向，等 M5 复盘时再判断。

## 7. 非目标

- 不做玩家移动。
- 不做开放大地图、房间、Chunk 或探索。
- 不做遗物、主动道具、升级选择、元进度。
- 不做完整本地化、设置系统、存档、回放、对象池矩阵。
- 不迁移到根目录 `client/`，直到明确决定把某个经验纳入完整项目。

## 8. 验收标准

- 可以启动 MVP 客户端并进入一个场景。
- 玩家固定在中心，方向键与手柄 D-pad / 摇杆能改变射击方向。
- 子弹按当前方向自动发射。
- 敌人从四方向出现并朝玩家推进。
- 有基本失败条件和重开方式。
- 制作过程至少记录 3 条经验或问题到 `MinimumViableProduct/docs/经验记录.md`。
