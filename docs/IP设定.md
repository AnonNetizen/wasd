## 1. 核心定位

**项目 IP 名**：破巢者

**英文暂定**：`Nestbreakers`

**核心句**：通道已经打开，防守只会慢慢死；真正能伤到敌人的办法，是派一名英雄杀进敌人的老家。

《破巢者》是一款多英雄固定斜俯视 2.5D 射击刷宝生存游戏。未知原因导致其他宇宙与本宇宙的通道突然打开，原本拥有银河系星际文明的人类被跨宇宙敌群打得七零八落，只剩首都星域和少数核心据点受到影响较轻。玩家每局选择一名英雄主动突入敌方“巢”的腹地，在无尽守军、机关和精英围攻中用主武器和射击强化技能打穿防线，刷取装备、词条、遗物、技能资源和战利品，升级构筑，并尝试打穿巢核；玩法行动仍在 2D 平面内判定，画面参考《哈迪斯》的斜俯视舞台表达，相机保持屏幕水平和等比缩放，用地面、障碍、遮挡层级和低模角色建立空间感。

## 2. “巢”的定义

“巢”不是单指虫巢，而是跨宇宙敌方在本宇宙建立或拖拽进来的核心据点、生产源头、通道锚点或意志中枢。

它可以表现为：

- 恶魔魔城
- 机械母厂
- 亡灵王庭
- 寄生森林
- 星舰核心
- 邪教圣所
- 深层矿井
- 黑暗炉心
- 界门堡垒
- 维度锚井

设计约束：后续敌方主题可以各自拥有强烈美术风格，但玩法包装统一为“破巢行动”。不要把“巢”锁死成虫族生态，也不要把每个主题拆成完全不同的系统入口。

## 3. 世界规则

本宇宙曾经拥有横跨银河系的星际文明：贸易航道、殖民星、轨道城市、军团舰队和远程通讯网络共同维持秩序。通道打开后，敌方并不是从一个方向入侵，而是在多个星区同时出现，把城市、舰队、行星设施和边境殖民地切成互不支援的孤岛。

首都星域因为拥有最完整的护盾、舰队、情报网和旧时代技术，仍能维持相对稳定。它不是安全天堂，而是最后还能组织反击的中心。其他星区有的已经沦陷，有的正在被巢化，有的只剩失联信标和求救记录。

敌方通过“巢”固定通道、生产守军、改造环境、释放机关、召集精英，并把周围空间变成适合自身宇宙规则运转的腹地。只在我方宇宙防守，最多拖慢扩张，无法真正削弱敌人的源头。

破巢者小队的策略是反过来突入源头：每次派出一名英雄，从外围切入巢内，在怪潮中越杀越深，夺取巢内遗物，把敌人的力量变成自己的构筑，并尝试破坏巢核、切断通道或让敌方战区失去供给。

死亡不是叙事断点，而是失败推进。英雄小队会带回样本、战利品和情报，用于局外成长、解锁新英雄、新遗物池、新巢类型和更高危险等级。

## 4. 玩家身份

玩家不是普通幸存者，而是首都星域下属破巢者组织的指挥者 / 行动记录者。每局派出一名英雄执行孤身突入。

多英雄的叙事重点是“为什么这个英雄敢独自进巢”：

- 复仇：家园星、舰队或亲友被敌方宇宙吞掉
- 赎罪：曾经参与通道研究、军事误判或撤离失败
- 救人：目标被敌方带入巢内，常规救援无法抵达
- 悬赏：首都星域以战利品、身份、赦免或星区通行权招募亡命者
- 信仰：认为防守无用，必须从敌方腹地斩首源头
- 实验：以自身能力、外骨骼、AI 或改造体验证破巢技术
- 诅咒：自身正在被异宇宙规则改写，必须杀到源头
- 职责：作为军人、特工、骑士、猎杀员或组织成员执行高危任务

## 5. 英雄方向

英雄是长期内容扩展的核心载体。每个英雄都应有身份、战斗定位、核心技能和构筑倾向。

推荐分类：

| 类型 | 包装方向 | 玩法倾向 |
|------|----------|----------|
| 先锋型 | 破阵、硬冲、前线队长 | 高生命、近身压迫、容错 |
| 枪械型 | 枪手、游侠、火力专家 | 射速、穿透、暴击、弹幕 |
| 工程型 | 工程师、装置师、无人机操作者 | 炮塔、无人机、陷阱、拾取强化 |
| 秘术型 | 术士、祭司、诅咒研究者 | 诅咒、召唤、状态效果、范围爆发 |
| 守护型 | 医师、盾卫、救援者 | 护盾、回复、反伤、持续作战 |
| 异化型 | 实验体、半巢化者、禁忌载体 | 低血爆发、副作用换强度、吞噬流 |
| 星舰型 | 舰队军官、坠舰幸存者、轨道火控员 | 标记、轰炸、无人支援、区域压制 |

英雄技能可以叫“主动技能”“专属天赋”或“终极技”。如果需要更强 IP 味，可在文案层使用“破巢技”，但系统命名仍应保持清楚，避免给玩家制造学习成本。

工程约束：英雄技能可以有强个性，但实现上应优先拆成可复用 `capability`、`effect`、`behavior`、`StatusEffect` 或 `ModifierEngine` primitive，禁止按英雄 id 写一次性特殊分支。

## 6. 敌方势力与巢主题

“巢”是统一玩法概念，具体主题可以多样化。

首批可扩展方向：

| 巢主题 | 敌方想象 | 适合敌人 | 适合 Boss / 巢核 |
|--------|----------|----------|------------------|
| 黑堡巢 | 从异宇宙压入的恶魔堡垒、军团指挥链 | 战士、盾兵、法师、精英将领 | 黑堡领主、军团核心 |
| 铁炉巢 | 改写物理规则的机械母厂、熔炉生产线 | 无人机、炮台、装甲体、维修单元 | 铁炉主脑、熔炉心脏 |
| 骨庭巢 | 死亡规则外溢的亡灵王庭、骨塔、死灵贵族 | 骷髅军、幽魂、骨兽、死灵法师 | 骨王、死庭王座 |
| 肉林巢 | 生物宇宙伸入本宇宙的寄生森林、孢子生态 | 孢子体、触手、再生怪、虫群 | 肉林母树、寄生核心 |
| 星舰巢 | 被敌方夺取或拖入通道的坠落星舰、能量舱 | 安保单元、异星士兵、能量无人机 | 舰载中枢、星核反应堆 |
| 界门巢 | 通道节点、维度锚井、敌方补给门 | 传送体、守门者、空间裂变体 | 界门主锚、通道心脏 |

## 7. 遗物包装

遗物是“巢内力量的碎片”。它们可能是敌方宇宙的武器模块、被污染的星际技术、通道锚点的副产物，或被巢核改写后的战利品。英雄夺取遗物后，敌人的规则会被反过来用于玩家构筑。

可用名称：

- 巢遗物
- 战利遗物
- 巢核碎片
- 源质样本
- 敌巢战利品

当前推荐：系统层沿用“遗物”，包装层可根据语境使用“巢遗物”或“战利遗物”。

命名原则：

- 中文短、有物感，尽量 2~5 字：`裂壳弹匣`、`赤炉核心`、`骨王戒指`
- 英文清楚、利于本地化：`Cracked Magazine`、`Red Furnace Core`、`Bone King's Ring`
- 描述要直接说功能，不把规则藏进设定：例如“子弹穿透 +1”，不要只写“它记得如何撕开甲壳”

## 8. 局外成长包装

基地可以称为：

- 破巢营地
- 前线营地
- 破巢者基地
- 首都星港
- 首都防线

局外资源可以包装为：

- 巢核碎片
- 战利样本
- 源质
- 巢烬

当前推荐：**巢核碎片**。它和标题绑定，也解释为什么失败后仍能推进：英雄虽未打穿巢核，但带回了核心样本、战利数据、通道参数或破巢情报。

局外成长可包括：

- 强化英雄基础属性
- 解锁新英雄
- 解锁新遗物池
- 解锁新巢主题
- 解锁开局选择
- 解锁更高危险等级

## 9. 命名与本地化

中文命名要优先直觉，避免过度谜语化；英文命名要清楚、短、方便 Steam 页面和商店搜索。

推荐词族：

| 中文 | 英文方向 | 用途 |
|------|----------|------|
| 破巢者 | Nestbreakers | IP / 组织 / 英雄群体 |
| 破巢行动 | nestbreaking run, nestbreak operation | 单局行动包装 |
| 敌巢 | enemy nest, stronghold, lair | 地点 / 主题 |
| 巢核 | nest core, enemy core | Boss / 终点 / 资源来源 |
| 巢遗物 | nest relic, enemy relic | 遗物包装 |
| 巢核碎片 | nest-core shard | 局外资源包装 |
| 首都星域 | capital sector, capital star sector | 我方相对稳定的最后中心 |
| 界门 | dimensional gate, inter-universe gate | 其他宇宙与本宇宙之间的通道 |

玩家可见正式文本必须进入 `client/locale/strings.csv`，并同时提供 `zh_CN` / `en`。本文件只定义命名方向，不替代 locale 表。

## 10. 宣传语方向

短句：

- 一个英雄，一座敌巢，无尽怪潮。
- 选择一名英雄，杀进敌人的老家。
- 夺取遗物，打穿巢核。
- 不是防守世界，而是反攻源头。
- 银河已经碎了，首都还在反击。
- 通道打开之后，防线只剩一个方向：向敌人深处推进。

英文方向：

- One hero. One enemy nest. Endless hordes.
- Break into the enemy stronghold. Steal their power. Shatter the core.
- Stop surviving. Start invading.
- The galaxy is broken. The capital still fights.

## 11. 法律与商标注意

“破巢者 / Nestbreakers”当前只是项目内部 IP 方向。此前轻量公开检索未发现同类游戏主标题的明显红旗，但英文侧存在若干 `Nestbreaker` / `Nest Breaker` 的任务、活动、物品或社区用法。

正式商用前必须做商标和商店名清检，至少覆盖：

- USPTO
- CNIPA
- WIPO Global Brand Database
- EUIPO
- Steam
- App Store / Google Play
- itch.io
- 域名和主要社媒账号

清检通过前，不要把英文名视为最终发行商标。
