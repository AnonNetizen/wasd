# 文档健康检查

> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。
> 本文档是文档健康检查范围、命令和失败解释的权威说明；改检查脚本、CI docs-check、知识库索引规则、`current_state.json` 校验或 `# Doc:` 覆盖规则时，必须同步 `docs/CICD规划.md`、`docs/AI知识库索引.md`、`docs/_kb_index.json`、`docs/AI记忆/current_state.json` 与当日会话日志。

---

## 1. 目标

把 AI 知识库从“靠约定维护”推进到“可校验”。当前健康检查覆盖无需 Godot、无需代码落地即可验证的文档结构；数据 / 契约检查由 `tools/sync_contracts.py --check` 与 `tools/validate_data.py` 配套执行。

## 2. 命令

```powershell
python tools/docs_health_check.py
python tools/sync_contracts.py --check
python tools/validate_data.py
```

成功时分别输出 `docs health check passed`、`sync contracts check passed`、`data validation passed`。

失败时输出具体错误并返回非零退出码。

## 3. 当前检查项

| 检查项 | 失败含义 | 修复方式 |
|--------|----------|----------|
| `docs/_kb_index.json` 是合法 JSON | 机器索引无法读取 | 修 JSON 语法 |
| 机器索引 schema 字段存在 | 索引格式漂移 | 补 `schema_version` / `documents` / `owner_scope` / `canonical_for` 等字段 |
| 索引中 `path` 存在 | 文档或目录被改名 / 删除后未同步索引 | 修路径或同步索引 |
| 索引中 `related_docs` 存在 | 联动文档路径失效 | 修路径或删除过期引用 |
| 索引中 `related_code` 存在 | 代码路径已删除或登记了尚未落地的路径 | 修路径；未落地能力写入说明而不是 `related_code` |
| 长期 Markdown 被索引覆盖 | 新文档未登记，AI 检索可能漏掉 | 在 `_kb_index.json` 登记文件或覆盖它的目录 |
| 长期 Markdown 顶部有 AI 修改说明 | 新文档缺少维护说明，或说明藏在正文中 | 在标题后 8 行内补 AI 修改说明 |
| Markdown 相对链接存在 | 文档死链或中文路径链接写错 | 修相对路径；中文文件名可直接写真实路径 |
| `docs/代码/README.md` 登记的模块文档存在 | 模块文档索引漂移 | 修索引或补文档 |
| `# Doc:` 指向存在 | 代码文件头引用失效 | 修路径或补模块文档 |
| ADR 编号递增且不重复 | 决策记录断裂 | 修 ADR 表格编号 |
| ADR 追踪矩阵覆盖最新 ADR | 新决策没有进入知识库追踪矩阵 | 更新 `docs/AI知识库索引.md` 的 ADR 追踪矩阵 |
| `current_state.json` 合法且字段齐全 | 机器当前状态不可读 | 补 `schema_version`、`latest_adr`、`open_decisions`、`next_actions`、`last_verified` 等字段 |
| `current_state.latest_adr` 等于 ADR 最新编号 | 当前状态与决策记录漂移 | 同步 `docs/AI记忆/current_state.json` |
| `current_state.open_decisions` 等于 `docs/修改建议.md` | 当前状态与待决策项漂移 | 同步 `current_state.json` 或 `docs/修改建议.md` |
| `current_state.next_actions` / `last_verified` 非空 | 新会话无法直接续接或不知道最近验证 | 写入下一步候选与最近验证命令 |

## 3-A. 配套数据 / 契约检查项

| 脚本 | 检查项 | 失败含义 |
|------|--------|----------|
| `tools/sync_contracts.py --check` | `docs/词表与契约.md`、`client/data/_contracts.json`、`client/scripts/contracts/*.gd` 是否同步 | 词表改了但未重生成，或生成文件被手改 |
| `tools/validate_data.py` | JSON 语法、`player.json`、`meta_progression.json`、`strings.csv`、MVP `mvp_config.json` | 数据字段类型 / 范围错、未登记 id、locale key 缺失或占位符不一致 |

## 4. 报告项（不阻塞）

| 报告项 | 含义 | 处理建议 |
|--------|------|----------|
| 孤儿 Markdown | 没有任何其他长期文档或 `_kb_index.json` 关联它 | 判断是否应补链接、登记到任务路由，或保留为独立入口 |
| 权威文档变更同步风险 | `authority` / `entrypoint` / `machine_index` 文件变了，但相关文档未同时变更 | 复查 `related_docs`，确认是否漏同步；若确实无需同步，可忽略报告 |

## 5. 暂不检查

| 暂不检查项 | 原因 | 后续落点 |
|------------|------|----------|
| 更细的内容 JSON Schema | 当前先由 `tools/validate_data.py` 覆盖已落地数据文件 | 后续为 `relics.json` / `enemies.json` 等补专项 schema |
| 代码文档覆盖率 | 正式 `client/` 代码尚未落地 | CI 阶段 2.H+ |
| 文档 frontmatter | 当前采用 `docs/_kb_index.json` 集中索引 | 需要时再迁移 |

## 6. 何时运行

- 新增、删除、重命名长期文档后。
- 修改 `docs/_kb_index.json` 后。
- 修改 `docs/代码/README.md` 或模块文档后。
- 修改脚本文件头 `# Doc:` 后。
- 修改 ADR 后。
- 修改 `docs/AI记忆/current_state.json`、`docs/AI记忆/项目记忆.md` 或 `docs/修改建议.md` 后。
- 修改三平台 `/health-check` 命令后。
- 修改 `docs/词表与契约.md`、`client/data/*.json`、`client/locale/strings.csv` 或 `MinimumViableProduct/client/data/*.json` 后。
- 提交前或创建 PR 前。

## 7. 与 CI 的关系

当前脚本可本地运行；CI 已在 `.github/workflows/docs-check.yml` 接入 `sync_contracts.py --check`、`validate_data.py` 与 `docs_health_check.py`。

CI 失败信息必须保持可读，方便 AI agent 根据报错直接修复。
