# 工具适配指南 —— 如果你不用 CodeBuddy

> 本项目所有规则、记忆、agents、commands 默认是为 **CodeBuddy IDE** 设计的。
> 但**核心资产 95% 与工具无关**——任何能读 markdown 的 AI 都能用本项目，只需对"自动加载机制"做一次性适配。
>
> 本文件告诉你：换别的 AI 工具时，**怎么让它和 CodeBuddy 用户一样有完整上下文**。

---

## 🎯 先记住一条万能兜底

不论换什么工具，**每次新对话开头说一句**：

> 先读 `AGENTS.md`。

`AGENTS.md` 是业界 AI IDE 通用约定。Aider / Claude Code / Cursor 新版 / Windsurf 等会**自动**读；不自动读的（ChatGPT / Copilot Chat / Claude Web 等），你手动提一句也立即生效。

`AGENTS.md` 已写明开工 5 步，AI 顺着就能找到规则、记忆、词表、测试策略——和工具原生支持哪种 frontmatter 完全无关。

## 🛠️ 各工具具体配法

### A. Claude Code（Anthropic 官方 CLI）

**最容易**——subagents/commands 格式与本项目几乎一致。

```powershell
# Windows PowerShell（建符号链接，无成本同步）
New-Item -ItemType SymbolicLink -Path .claude -Target .codebuddy
```

```bash
# Linux / macOS
ln -s .codebuddy .claude
```

或直接复制：

```bash
cp -r .codebuddy .claude          # Linux/macOS
xcopy /E /I .codebuddy .claude    # Windows cmd
```

再让 Claude Code 看到入口：

```bash
echo "请先阅读 AGENTS.md 与 .claude/rules/game-coding-rules.md，再开始任务。" > CLAUDE.md
```

完成。从此 Claude Code 用户体验与 CodeBuddy 等同。

> 注：如果 Claude Code 版本支持 frontmatter 的 `tools` 字段差异（CodeBuddy 用 `read_file`/`search_content`，Claude Code 工具名可能不同），按报错信息改两个名字即可。

---

### A2. OpenAI Codex CLI（零成本，已镜像）

**本项目已内置完整 `.codex/` 镜像**，结构与 `.codebuddy/` 完全一致：

- `.codex/agents/` — 3 个 subagents（`balancer` / `contract-validator` / `data-author`）
- `.codex/commands/` — 5 个 slash commands（`health-check` / `new-relic` / `run-replay-regression` / `sync-contracts` / `update-memory`）
- `.codex/rules/game-coding-rules.md` — 完整编码规则（含双平台同步约束）

Codex CLI 会自动读取 `.codex/` 下的规则与配置。你无需做任何额外设置。

> **双平台同步规则（规则 21）**：修改 `.codebuddy/` 或 `.codex/` 任一文件夹中的文件时，**必须同步修改另一个**。两个目录完全镜像，禁止只改一边。pre-commit hook 与 CI `health-check` 会校验一致性。

### B. Cursor

`.cursor/rules/*.mdc` 启用，但 frontmatter 字段不一样。**最省事做法**：建一个 stub 转发到原文件。

```bash
mkdir -p .cursor/rules
```

新建 `.cursor/rules/main.mdc`：

```markdown
---
description: 主项目规则
globs: ["**/*"]
alwaysApply: true
---

完整规则见 [.codebuddy/rules/game-coding-rules.md](../../.codebuddy/rules/game-coding-rules.md)。

开工前必读：
- AGENTS.md
- docs/AI记忆/项目记忆.md
- docs/AI导航.md
```

Cursor 会把这份 stub 一直挂在上下文，里面的相对链接 Cursor 也会顺着读。

> Subagents/commands 在 Cursor 里没原生入口，但你可以把 `.codebuddy/agents/balancer.md` 当 prompt 片段——`@balancer.md` 引用即可。

### C. Aider

仓库根加 `.aider.conf.yml`：

```yaml
# Aider 启动时自动加载到上下文
read:
  - AGENTS.md
  - .codebuddy/rules/game-coding-rules.md
  - docs/AI记忆/项目记忆.md
  - docs/AI导航.md

# 提交风格（与本项目 .gitmessage 对齐）
commit-prompt: |
  按 Conventional Commits 风格写 commit。type 用 feat/fix/docs/data/locale/refactor/perf/style/chore/ci/test/revert。
```

完成。

### F. GitHub Copilot Chat（无项目级规则系统）

没有自动加载机制。**每次 chat 开头说**：

> 先读 `AGENTS.md`，再帮我做 ...

或在 VSCode 设置里加自定义 system prompt（如果你的 Copilot 版本支持 `github.copilot.chat.codeGeneration.instructions`）：

```jsonc
"github.copilot.chat.codeGeneration.instructions": [
  { "file": "AGENTS.md" },
  { "file": ".codebuddy/rules/game-coding-rules.md" }
]
```

---

### G. ChatGPT / Claude Web / 通用 AI 聊天框

把这段话**复制**到对话开头：

```
我在做一个 Roguelike 游戏项目（Godot 4.6.3 + GDScript），仓库地址是 <你的 repo URL>。

请先按以下顺序读 5 个文件，再开始任何任务：
1. AGENTS.md
2. docs/AI记忆/项目记忆.md
3. .codebuddy/rules/game-coding-rules.md
4. docs/AI导航.md
5. 按当前任务读对应模板：docs/AI协作/任务模板/

如果你无法直接读取仓库文件，告诉我，我把这几份文件内容贴给你。
```

仓库公开的话，多数模型能直接抓 GitHub raw URL；不行就贴文件内容。

## 🤖 关于 subagents 与 slash commands

它们是 CodeBuddy / Claude Code 等少数工具的"调度糖"。**不支持也没关系**：

- `.codebuddy/agents/*.md` 同时是**精确的角色 prompt 模板**
- `.codebuddy/commands/*.md` 同时是**精确的操作步骤手册**

所以哪怕你的工具不识别它们，要"跑平衡分析"时直接说：

> 按 `.codebuddy/agents/balancer.md` 的角色定位帮我跑回放回归。

要"同步词表常量"时直接说：

> 按 `.codebuddy/commands/sync-contracts.md` 的步骤跑同步。

效果一样——只是少了 `/` 这个快捷键。

## 📋 一次性适配 checklist

不论换什么工具，做这 4 步：

- [ ] 仓库根有 `AGENTS.md`（已有 ✅）
- [ ] 你的 AI 工具能"自动读 AGENTS.md"或你愿意每次手动提一句
- [ ] （可选）按上面 A~G 节做工具专属的规则加载配置
- [ ] 跑一次冷启动测试：新开一个对话，让 AI 描述本项目当前状态——它应当能答出"v1.5、30 条 ADR、文档阶段、下一步是 M1"

如果第 4 步 AI 答得出，说明上下文加载成功；答不出就是规则文件没被读到，回去检查工具配法。
