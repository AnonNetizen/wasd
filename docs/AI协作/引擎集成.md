# 引擎集成（MCP / Bridge 接入指南）

> 现状：AI agent 默认只能"读写文件"。当引擎 MCP/Bridge 可用时，AI 可直接查询场景树、读 Resource、触发引擎命令——这是"AI 协作"的能力跃升。
> 本指南列出可接入的工具与配置方式，帮助 agent 与人决定**何时启用、如何启用**。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是引擎集成指南；新增 MCP / Bridge 工具或改变使用优先级时，必须同步 `docs/AI协作/工具适配指南.md`、`docs/AI导航.md`、规则 14-A，必要时更新 README / CONTRIBUTING。

## 1. Godot MCP（推荐关注）

社区已有多个开源实现，搜索关键词：`godot-mcp`、`godot mcp server`。

### 能力（典型）
- 读项目树（`res://` 下所有资源）
- 读场景节点结构（`.tscn` 解析）
- 读/写 `Resource`（`.tres`）
- 触发 `godot --headless` 跑特定场景
- 编辑器在线查询当前选中节点（需编辑器扩展）

### 何时启用
- AI 在大型场景树中找节点路径痛苦时
- 需要批量改 `.tres` Resource 时
- 跑 headless 测试需要交互式调试时

### 何时不启用
- 单纯的脚本 / JSON 改动 —— 文件操作够用，多一层 MCP 反而增加不确定性
- 任务能用本目录的"任务模板"覆盖时

### 配置位（待实际接入时填入）
```
TODO（接入后补充）：
- 安装步骤
- codebuddy mcp 配置示例（mcp.json 节选）
- 与项目的对接（启动脚本、端口、白名单工具等）
```

## 2. Unity MCP（备选，本项目暂不用）

社区也有 Unity MCP 实现，能力对齐 Godot 版（场景查询 / Prefab 操作 / 触发 Play 模式等）。本项目用 Godot，仅做记录，未来若多引擎并行再启用。

## 3. Aider / Continue / Cursor 风格的"实时验证"

并不是 MCP，但属于"引擎之外"的 AI 加速器：
- **Aider**：Git diff 驱动的 AI 编辑，可与 pre-commit hook 互补
- **Continue / Cursor**：IDE 级 AI，能实时跑 lint/test
- 本项目的 `实时验证回路.md` 已内建等效的本地反馈，**不强依赖 IDE**

## 4. 自研轻量 Bridge（正式 client 默认）

本项目先采用 `tools/godot_bridge.py` 作为轻量 Bridge，不依赖 MCP 协议；它把 Godot / 数据校验 / 场景树导出封装为稳定命令，供 AI 和 CI 前置验证使用。

当前命令：

```powershell
python tools/godot_bridge.py export-tree
python tools/godot_bridge.py validate-data
python tools/godot_bridge.py godot-version
python tools/godot_bridge.py headless-boot
python tools/godot_bridge.py runtime-smoke
python tools/godot_bridge.py settings-smoke
python tools/godot_bridge.py gear-mod-smoke
python tools/godot_bridge.py save-smoke
python tools/godot_bridge.py --project client module-world-smoke
python tools/godot_bridge.py --project client module-world-technical-slice-smoke
python tools/godot_bridge.py --project client startup-probe
```

默认项目路径为正式 `client/`；需要指定其他 Godot 项目时使用：

```powershell
python tools/godot_bridge.py --project path/to/project headless-boot
```

`headless-boot` 会先用 `--headless --editor` 完成资源导入与全局 `class_name` 扫描，再启动正式主场景；任一阶段出现脚本解析错误都会返回失败。这样新 clone 即使尚未生成 `.godot/global_script_class_cache.cfg`，验证结果也不会依赖本机编辑器缓存。

当前能力：

- `export-tree`：解析 `scenes/*.tscn`，输出场景节点树 JSON（不启动 Godot）。
- `validate-data`：调用 `tools/validate_data.py`。
- `godot-version`：调用本机 Godot 输出版本。
- `headless-boot`：运行 `godot --headless --path <project> --quit`。
- `runtime-smoke`：运行正式主场景并传入 `--runtime-smoke`，通过 `--open-warzone` 非默认回归 carrier 继续断言 F4/F12 输入、池化、伤害、兴趣点和结算。
- `settings-smoke`：运行正式主场景并传入 `--settings-smoke`，由 `client/tools/settings_smoke.gd` 断言 F7 设置持久化、范围校验、坏配置回退和语言设置联动。
- `gear-mod-smoke`：运行正式主场景并传入 `--gear-mod-smoke`，由 `client/tools/gear_mod_smoke.gd` 断言 F11 装备 Mod profile、装配、升级、分解和掉落。
- `save-smoke`：断言 run v4 roundtrip、`.bak` 回退、双坏档隔离和 v1→v2→v3→v4 迁移；v3 run 被明确标记不兼容，`meta` 不受影响。
- `module-world-smoke`：断言 F13 同 seed assignment / 内容敏感 hash、不同 seed 变化、坐标转换、物理移动无缝跨边界、3×3 chunk 上限、离开 / 返回不重复生成、子弹 / 掉落按槽位释放恢复、迷雾、目标后撤离、run v4 恢复与坏 hash fail closed。
- `module-world-technical-slice-smoke`：经 `--module-world-technical-slice` opt-in 入口跑中心 3×3 首片，断言外圈 72 槽封锁及片内目标—撤离 / 续局。
- `startup-probe`：仅在用户明确要求性能测试时，以 Bridge 的单调时钟测量正式主场景 `BOOT_BEGIN` 首帧到默认模块世界进入 `PLAYING` 的耗时，硬门槛为 2 秒；进程冷启动到 `BOOT_BEGIN` 只作诊断。该命令不属于日常回归或默认交付义务。

后续扩展：

- `--play-replay <file>`：headless 跑回放并输出帧 diff。
- `--validate-project`：完整项目落地后校验 autoload、Project Settings 和资源引用。

> 推荐：**项目早期先做自研 bridge**（成本低、可控），等 Godot MCP 生态成熟再换。

## 5. 元规则

- 任何引擎集成方式的启用，应在 `决策记录.md` 加 ADR，并在 `项目记忆.md` 第 5 节"工具链与基础设施"标注。
- AI agent 工作时**先确认当前是否启用了 MCP/Bridge**，避免重复造轮子。
