# 实时验证回路（本地秒级反馈）

> 业界 AI 协作工作流（Cursor / Aider / Continue 等）能力的核心：**改完代码秒级知错**。
> 本项目通过 pre-commit hook + watch 脚本，把 CI 的核心校验前置到本地，使 AI agent 在 commit 前就被打回，避免"push → CI → 等待"的分钟级回路。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是本地验证回路权威；改 hook / watch / 本地校验职责时，必须同步 `docs/CICD规划.md`、`docs/测试策略.md`、`CONTRIBUTING.md`、规则自检清单。

## 1. 三层反馈

| 层级 | 反馈延迟 | 工具 | 检查内容 |
|------|---------|------|---------|
| **保存即检（毫秒~秒）** | 编辑器内 | `.editorconfig` + IDE lint 插件 + watch 脚本 | 缩进/换行/未类型化变量 |
| **commit 前（秒级）** | pre-commit hook | 见下表 | 数据 schema / 词表契约 / 文档链接 |
| **push 后（分钟级）** | CI workflow | `docs/CICD规划.md` 阶段 1 | 全量校验 + 跨文件检查 |

理想情况：CI 是"防线兜底"，commit hook 是"日常拦截"，IDE 是"打字时提醒"。

当前落地状态：已启用 `.github/workflows/docs-check.yml` 作为最小 CI 防线，包含契约同步、数据 / locale 校验、DataLoader schema 回归、第一档 GDScript 项目 lint、第二档项目规则 lint、第三档语义 advisory lint、文档健康和 whitespace；仓库已提供 `.pre-commit-config.yaml`，本地安装后可在 commit 前跑同一批 Stage 1 检查，追加不启动 Godot 的 Steamworks Lab toolchain 单元回归，并在模块 JSON / Dock / catalog / baker / 生成场景相关文件变化时条件式运行只读 `module-bake-check`，同时对 staged 文件的 EOF 多空行做自动修复。

## 2. pre-commit hook（已落地）

工具：[`pre-commit`](https://pre-commit.com/)（Python 框架，`.pre-commit-config.yaml` 配置）。

安装：

```powershell
pip install pre-commit
pre-commit install
```

手动运行：

```powershell
pre-commit run --all-files
```

如果当前机器没有安装 pre-commit，按下表的脚本逐个运行即可。

**框架阶段已配置的 hook**：

| hook | 脚本 | 作用 |
|------|------|------|
| `sync-contracts-check` | `tools/sync_contracts.py --check` | 解析 `词表与契约.md`，确认 `_contracts.json` 与 GDScript 常量未漂移 |
| `validate-data` | `tools/validate_data.py` | 校验 `client/data/*.json` / `client/data/*.csv`、locale CSV 中的 schema、词表 id 与 locale key |
| `test-data-loader-schema` | `tools/test_data_loader_schema.py` | 用临时数据副本断言 DataLoader-facing schema 黄金数据通过、坏 id / 缺 locale / 错类型 / 错范围 / 跨文件引用错失败 |
| `lint-gdscript-rules` | `tools/lint_gdscript_rules.py` | 第一档 GDScript 项目规则 lint：段落顺序、危险 `:=`、中文硬编码字符串、裸随机 / 时间 / 暂停 API |
| `lint-project-rules` | `tools/lint_project_rules.py` | 第二档项目规则 lint：数据字段手册覆盖、locale 双语、release preset 禁 debug/dev_tools |
| `test-project-rules-lint` | `tools/test_project_rules_lint.py` | 固定项目规则 lint 的坏样例，防止字段文档、双语和 release 检查退化 |
| `lint-semantic-rules` | `tools/lint_semantic_rules.py` | 第三档非阻塞语义 advisory：特殊 id 分支、autoload 绕过、类型签名、`# Doc:`、contract 常量引用风险；Pool / UI 提示命名可识别的高频实体 / 弹窗风险及正式 gameplay/UI 的长期 Node/Control `.new()` 挂树，并豁免已注册 factory 与可编辑行模板实例化 |
| `test-semantic-rules-lint` | `tools/test_semantic_rules_lint.py` | 固定语义 advisory lint 的报警与不误报样例，防止 warning 规则退化或重新泛化 |
| `test-steamworks-lab-toolchain` | `tools/test_steamworks_lab_toolchain.py` | 固定 console 解析、独立用户环境、精确成功 / 错误标志、超时终止、动态端口、轻量临时副本和源码 `steam_appid.txt=4955670` 保护 |
| `module-bake-check` | `tools/godot_bridge.py --project client module-bake-check` | 仅相关路径变化时启动 Godot，无写入检查 JSON v2、tile catalog、审核 hash 与全部生成 TSCN 的规范指纹 |
| `docs-health-check` | `tools/docs_health_check.py` | 校验 `docs/_kb_index.json`、`current_state.json`、长期文档覆盖、Markdown 链接、`related_code`、AI 修改说明位置、ADR 矩阵、`# Doc:` 引用和模块文档索引，并输出孤儿文档 / 同步风险报告 |
| `staged-whitespace-check` | `tools/check_staged_whitespace.py` | 自动修复 staged 文件的 EOF 多空行，再对 staged diff 跑 whitespace 检查，排除 `draft/` / `DRAFT/`；其他 whitespace 错误仍失败 |

## 4. AI agent 行为约定

- **agent 改完文件应自动跑 hook**：执行 `pre-commit run --all-files` 或 `pre-commit run --files AGENTS.md docs/AI导航.md`，再决定是否 commit；没安装 pre-commit 时跑等价命令。
- **失败时自我纠错**：hook 输出格式约定为「文件 + 字段路径 + 期望值」，agent 据此直接定位修复。
- **不要绕过**：禁止使用 `--no-verify` 跳过 hook（除非明确告知用户原因）。
- **review 先看工具输出**：正式 review 按 `docs/AI协作/代码审核流程.md` 执行，先处理 lint/test/docs check，再让 LLM 重点读当前 diff。

## 5. 与 CI 的关系

- CI（`docs/CICD规划.md` 阶段 1）跑**全量 + 跨文件**校验，是最终防线。
- 当前 CI 跑 `tools/sync_contracts.py --check`、`tools/validate_data.py`、`tools/test_data_loader_schema.py`、`tools/lint_gdscript_rules.py`、`tools/lint_project_rules.py`、`tools/test_project_rules_lint.py`、非阻塞 `tools/lint_semantic_rules.py`、`tools/test_semantic_rules_lint.py`、`tools/docs_health_check.py` 与 whitespace diff；commitlint、Godot headless 和 GUT 后续再加。
- pre-commit 对 Stage 1 校验保持全量执行；Godot `module-bake-check` 已按模块相关路径条件式触发，普通提交不启动 Godot。
- 两者用同一份脚本（`tools/*.py`），CI 调用方式不同：
  - CI：`python tools/sync_contracts.py --check && python tools/validate_data.py && python tools/test_data_loader_schema.py && python tools/lint_gdscript_rules.py && python tools/lint_project_rules.py && python tools/lint_semantic_rules.py`
  - hook：`.pre-commit-config.yaml` 调用同一批脚本，并额外运行 staged whitespace check

## 7. 元规则

- 任何 hook 新增/调整，需在 `决策记录.md` 加 ADR + 同步本文件 + 更新 `docs/CICD规划.md`。
- hook 失败但用户需绕过时，必须在 commit message 写明原因（`chore: bypass <hook> due to <reason>`）。
