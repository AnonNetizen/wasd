# 实时验证回路（本地秒级反馈）

> 业界 AI 协作工作流（Cursor / Aider / Continue 等）能力的核心：**改完代码秒级知错**。
> 本项目通过 pre-commit hook + watch 脚本，把 CI 的核心校验前置到本地，使 AI agent 在 commit 前就被打回，避免"push → CI → 等待"的分钟级回路。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是本地验证回路权威；改 hook / watch / 本地校验职责时，必须同步 `docs/CICD规划.md`、`docs/测试策略.md`、`CONTRIBUTING.md`、规则自检清单。

---

## 1. 三层反馈

| 层级 | 反馈延迟 | 工具 | 检查内容 |
|------|---------|------|---------|
| **保存即检（毫秒~秒）** | 编辑器内 | `.editorconfig` + IDE lint 插件 + watch 脚本 | 缩进/换行/未类型化变量 |
| **commit 前（秒级）** | pre-commit hook | 见下表 | 数据 schema / 词表契约 / 文档链接 |
| **push 后（分钟级）** | CI workflow | `docs/CICD规划.md` 阶段 1 | 全量校验 + 跨文件检查 |

理想情况：CI 是"防线兜底"，commit hook 是"日常拦截"，IDE 是"打字时提醒"。

当前落地状态：已启用 `.github/workflows/docs-check.yml` 作为最小 CI 防线；pre-commit hook 仍待实现，暂不要求本地自动安装。

## 2. pre-commit hook 设计（待实现）

工具：[`pre-commit`](https://pre-commit.com/)（Python 框架，`.pre-commit-config.yaml` 配置）。

**框架阶段就能跑的 hook**：

| hook | 脚本 | 作用 |
|------|------|------|
| `validate-contract` | `tools/validate_contract.py` | 解析 `词表与契约.md` 抽白名单，扫 `client/data/*.json` 中是否有未登记 id |
| `docs-health-check` | `tools/docs_health_check.py` | 校验 `docs/_kb_index.json`、`current_state.json`、长期文档覆盖、Markdown 链接、`related_code`、AI 修改说明位置、ADR 矩阵、`# Doc:` 引用和模块文档索引，并输出孤儿文档 / 同步风险报告 |
| `check-doc-links` | `tools/check_doc_links.py` | 扫所有 md 文件的相对链接，检查是否死链 |
| `check-numbering` | `tools/check_numbering.py` | `修改建议.md` 编号唯一、`决策记录.md` ADR 编号连续 |
| `check-version-sync` | `tools/check_version_sync.py` | `README.md` 与 `游戏设计文档.md` 版本号一致 |
| `commitlint` | `commitlint-precommit` | Conventional Commits 校验 |

**代码落地后追加**：

| hook | 脚本 | 作用 |
|------|------|------|
| `gdformat-check` | `gdformat --check` | GDScript 格式化 |
| `gdlint` | `gdlint` | GDScript 静态检查 |
| `validate-schema` | `ajv-cli` | 数据 JSON Schema 校验 |
| `typing-check` | 自定义 | 强制类型化 GDScript |

## 3. Watch 脚本（开发期可选）

跑 `tools/watch.py`（待实现）：监听 `docs/词表与契约.md` 与 `client/data/` 变化，**改完即跑**对应校验，结果打印到终端。

适合：批量改数据时的体验，不必每次 commit。

---

## 4. AI agent 行为约定

- **agent 改完文件应自动跑 hook**：执行 `git add ... && pre-commit run --files <changed>`，再决定是否 commit。
- **失败时自我纠错**：hook 输出格式约定为「文件 + 字段路径 + 期望值」，agent 据此直接定位修复。
- **不要绕过**：禁止使用 `--no-verify` 跳过 hook（除非明确告知用户原因）。

---

## 5. 与 CI 的关系

- CI（`docs/CICD规划.md` 阶段 1）跑**全量 + 跨文件**校验，是最终防线。
- 当前 CI 最小版只跑 JSON、CSV、`tools/docs_health_check.py` 与 whitespace diff；词表同步、commitlint、Godot headless 和 GUT 后续再加。
- pre-commit 跑**增量 + 单文件**校验，速度优先。
- 两者用同一份脚本（`tools/*.py`），CI 调用方式不同：
  - CI：`python tools/validate_contract.py --all`
  - hook：`python tools/validate_contract.py --files <changed>`

## 7. 元规则

- 任何 hook 新增/调整，需在 `决策记录.md` 加 ADR + 同步本文件 + 更新 `docs/CICD规划.md`。
- hook 失败但用户需绕过时，必须在 commit message 写明原因（`chore: bypass <hook> due to <reason>`）。
