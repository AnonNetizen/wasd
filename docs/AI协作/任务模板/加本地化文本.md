# 任务模板：加一段面向玩家的文本

> **AI 修改说明**：修改本任务模板前先读 `docs/AI协作/文档维护指南.md`。模板变化通常需要同步 `docs/AI协作/README.md`、`docs/AI协作/上下文预算.md`、相关 subagent / command（如有）、`AGENTS.md` 的高频任务表（如入口变化）。

## 必读
1. 当前平台编码规则入口（见 `AGENTS.md` 第 3 步）第 4 节（多语言本地化）
2. `docs/词表与契约.md` 第 6 节（本地化 key 命名规范）
3. `client/locale/README.md`（多语言文案配置手册）
4. `client/locale/strings.csv`

## 核心约束
- ❌ 绝不硬编码文本
- ✅ 一律用 `tr("key")` 或数据中的 `name_key` / `desc_key`
- ✅ key 命名遵循 `<域>_<对象>_<字段>` 蛇形小写
- ✅ 动态数值用占位符 `{value}`，禁止字符串拼接
- ✅ `zh_CN` 与 `en` 两列都要补齐，新增语言或占位符规则变化时同步 `client/locale/README.md`
- ✅ 用户只提供一种语言时，AI 自动生成另一语言首版译文；人工校对是最终权威

## 步骤
1. 起 key 名（按域前缀：`ui_` / `relic_` / `item_` / `enemy_` / `hint_`）
2. 在 `client/locale/strings.csv` 加一行：`key, zh_CN, en, ...`
3. 在使用处用 `tr("<key>")` 或在数据文件里填 `name_key` / `desc_key`
4. 至少补 `zh_CN` 与 `en` 两列；如果只有中文，AI 译英文；如果只有英文，AI 译中文；其他语言可暂时同 `en` 占位
5. 若新增语言列、key 域前缀或占位符规则，更新 `client/locale/README.md` 与词表

## 自检
- [ ] 没有任何裸字符串残留在代码 / 数据
- [ ] key 命名合规
- [ ] 至少 2 种语言
- [ ] AI 自动补译内容已标记为需要人工复核，且未改变原功能含义
- [ ] 所有语言的占位符集合一致
- [ ] hook 全过

## 不需要做的
- 不需 ADR
- 不需 AI 记忆同步
