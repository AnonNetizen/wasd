# 任务模板：加一段面向玩家的文本

## 必读
1. `.codebuddy/rules/game-coding-rules.md` 第 4 节（多语言本地化）
2. `docs/词表与契约.md` 第 6 节（本地化 key 命名规范）
3. `client/locale/strings.csv`

## 核心约束
- ❌ 绝不硬编码文本
- ✅ 一律用 `tr("key")` 或数据中的 `name_key` / `desc_key`
- ✅ key 命名遵循 `<域>_<对象>_<字段>` 蛇形小写
- ✅ 动态数值用占位符 `{value}`，禁止字符串拼接

## 步骤
1. 起 key 名（按域前缀：`ui_` / `relic_` / `item_` / `enemy_` / `hint_`）
2. 在 `client/locale/strings.csv` 加一行：`key, zh_CN, en, ...`
3. 在使用处用 `tr("<key>")` 或在数据 JSON 里填 `name_key` / `desc_key`
4. 至少补 `zh_CN` 与 `en` 两列；其他语言可暂时同 `en` 占位

## 自检
- [ ] 没有任何裸字符串残留在代码 / 数据
- [ ] key 命名合规
- [ ] 至少 2 种语言
- [ ] hook 全过

## 不需要做的
- 不需 ADR
- 不需 AI 记忆同步
