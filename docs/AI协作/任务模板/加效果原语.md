# 任务模板：加一种效果原语

> 这是**改逻辑层**的任务，比加遗物/数据严肃。**先登记，再实现，最后才有人能在数据里使用**。

## 必读
1. `.codebuddy/rules/game-coding-rules.md` 第 6/15 节（数据驱动 + 词表契约）
2. `docs/词表与契约.md` 第 2 节（effect 白名单）
3. 已有效果原语的实现（`client/scripts/effects/` 等）
4. `docs/决策记录.md`（看历史决策风格）

## 核心约束
- ✅ **先在词表登记**，再写代码
- ✅ 类型化 GDScript
- ✅ 实现为脚本方法或小型 Node/Resource
- ❌ 不在数据里直接使用尚未实现的 effect id（CI 会拦）

## 步骤
1. 在 `docs/词表与契约.md` 第 2 节加一行：`<id>` / 含义 / params 字段说明
2. 实现：在 `client/scripts/effects/` 加 `<id>.gd`，按既有原语风格写
3. 注册：在 `EffectRegistry` 或等价分发表里登记 id → 实现的映射
4. 写一条「黄金样例遗物」用此原语（或在已有遗物里引用）
5. 在 `决策记录.md` 加一条 ADR：「引入 `<id>` 原语；理由」
6. 在 `项目记忆.md` 第 3 节追加该 ADR 摘要
7. 跑 pre-commit + 项目启动校验

## 自检
- [ ] 词表已登记
- [ ] 代码类型化、复用模板
- [ ] 已有黄金样例引用
- [ ] ADR 已加
- [ ] AI 记忆已同步
- [ ] hook 全过

## 不需要做的
- 不需把所有现有遗物改去用这个原语（按需）
