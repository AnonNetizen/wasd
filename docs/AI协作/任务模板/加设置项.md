# 任务模板：加一个设置项

> **AI 修改说明**：修改本任务模板前先读 `docs/AI协作/文档维护指南.md`。模板变化通常需要同步 `docs/AI协作/README.md`、`docs/AI协作/上下文预算.md`、相关 subagent / command（如有）、`AGENTS.md` 的高频任务表（如入口变化）。

## 必读
1. 当前平台编码规则入口（见 `AGENTS.md` 第 3 步）第 5 节（设置系统）
2. `docs/词表与契约.md` 第 5 节（设置 key）
3. `client/scripts/settings.gd`（即 `Settings` autoload）
4. 设置 UI 场景（`client/scenes/ui/settings.tscn` 等）

## 核心约束
- ✅ 设置项 = 一条配置（键/类型/默认/范围）+ 一个 UI 控件
- ✅ 走 `setting_changed(key, value)` 信号即时生效
- ✅ 持久化到 `user://settings.cfg`
- ❌ 不在业务代码里直接读 ConfigFile，必须经 `Settings` 接口

## 步骤
1. 在 `docs/词表与契约.md` 第 5 节加一行：`<key>` / 类型 / 默认 / 取值范围
2. 在 `Settings.gd` 注册该项的默认值
3. 在设置 UI 场景里加一个控件，绑定该 key
4. 订阅方系统在 `_ready` 里 `Settings.connect("setting_changed", ...)`
5. 在 `client/locale/strings.csv` 加该控件的本地化标签

## 自检
- [ ] 词表已登记
- [ ] 默认值注册到 `Settings`
- [ ] UI 控件已加 + 本地化
- [ ] 订阅方即时生效
- [ ] hook 全过

## 不需要做的
- 不需 ADR（设置项扩展属于框架内行为）
- 不需 AI 记忆同步（除非是开关型重大改动）
