# 任务模板：加一个埋点事件

## 必读
1. `.codebuddy/rules/game-coding-rules.md` 第 13 节（数据埋点框架）
2. `docs/词表与契约.md` 第 4 节（埋点事件）
3. `client/scripts/analytics.gd`（即 `Analytics` autoload）

## 核心约束
- ✅ 事件名集中定义为常量/配置
- ✅ 通过 `Analytics.track_event(event_name, params)` 上报
- ❌ 不散落硬编码事件名 / 字段在业务代码

## 步骤
1. 在 `docs/词表与契约.md` 第 4 节加一行：`<event_name>` / 触发时机 / 关键字段
2. 在 `Analytics.gd` 的 `EventNames` 常量类（或等价位置）加常量
3. 在业务代码里调用：`Analytics.track_event(EventNames.<NAME>, { ... })`
4. 字段值如涉及位置/时间/版本，复用 `Analytics` 已有的辅助函数

## 自检
- [ ] 词表已登记
- [ ] 代码用常量引用，无裸字符串
- [ ] 字段集中定义
- [ ] hook 全过

## 不需要做的
- 不需 ADR（一般埋点扩展不需要）
- 接后端是后续话题，本任务只到「本地落盘可用」即结束
