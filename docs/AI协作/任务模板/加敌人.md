# 任务模板：加一个敌人

> **AI 修改说明**：修改本任务模板前先读 `docs/AI协作/文档维护指南.md`。模板变化通常需要同步 `docs/AI协作/README.md`、`docs/AI协作/上下文预算.md`、相关 subagent / command（如有）、`AGENTS.md` 的高频任务表（如入口变化）。

## 必读
1. 当前平台编码规则入口（见 `AGENTS.md` 第 3 步）第 6/10 节（数据驱动 + 性能）
2. `docs/词表与契约.md` 第 1 节（stat 名）
3. `client/data/README.md`（敌人字段、单位、范围说明）
4. `client/locale/README.md`（敌人名称 key 与语言列规则）
5. `client/data/enemies.csv` 黄金样例
6. `client/templates/enemy_template.gd`（落地后）
7. 已有的 `EnemyAI` 行为类型清单（落地后在 `client/scripts/enemy/`）

## 核心约束
- ✅ 优先复用既有 AI 类型（追击 / 远程 / 群集）
- ✅ 新行为才碰逻辑（先加 ADR）
- ✅ 高频敌人接对象池
- ✅ 名称 / 描述走 locale key，不在敌人数据里写硬文本；缺中文或英文时由 AI 自动补首版译文

## 步骤
1. 决定敌人类别 → 复用既有 AI 类型还是新建
2. 复用：直接在 `client/data/enemies.csv` 加一行，引用既有 pool id / 伤害类型 / 标签
3. 新建：先在 `决策记录.md` 加一条草稿 ADR → 实现新 AI 类型 → 再填数据
4. `client/locale/strings.csv` 加 `enemy_xxx_name`，至少补 `zh_CN` / `en`；用户只给一种语言时 AI 自动补另一种
5. 美术阶段先用占位几何形状

## 自检
- [ ] AI 类型复用既有的（如果新建，已有 ADR）
- [ ] stat id 都在词表
- [ ] locale key 已按 `client/locale/README.md` 补齐
- [ ] AI 自动补译未改变敌人定位或功能含义
- [ ] 接入 `Spawner` 数据（如有需要）
- [ ] 经过 hook 校验

## 不需要做的
- 不需写 `enemies.csv` 之外的数据文件
- 不需改 `Spawner` 代码（除非新建波次类型）
