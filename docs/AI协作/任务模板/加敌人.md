# 任务模板：加一个敌人

## 必读
1. `.codebuddy/rules/game-coding-rules.md` 第 6/10 节（数据驱动 + 性能）
2. `docs/词表与契约.md` 第 1 节（stat 名）
3. `client/data/enemies.json` 黄金样例
4. `client/templates/enemy_template.gd`
5. 已有的 `EnemyAI` 行为类型清单（在 `client/scripts/enemy/`）

## 核心约束
- ✅ 优先复用既有 AI 类型（追击 / 远程 / 群集）
- ✅ 新行为才碰逻辑（先加 ADR）
- ✅ 高频敌人接对象池

## 步骤
1. 决定敌人类别 → 复用既有 AI 类型还是新建
2. 复用：直接在 `client/data/enemies.json` 加一条，引用既有 AI 类型 id
3. 新建：先在 `决策记录.md` 加一条草稿 ADR → 实现新 AI 类型 → 再填数据
4. `client/locale/strings.csv` 加 `enemy_xxx_name`
5. 美术阶段先用占位几何形状

## 自检
- [ ] AI 类型复用既有的（如果新建，已有 ADR）
- [ ] stat id 都在词表
- [ ] 接入 `Spawner` 数据（如有需要）
- [ ] 经过 hook 校验

## 不需要做的
- 不需写 `enemies.json` 之外的数据文件
- 不需改 `Spawner` 代码（除非新建波次类型）
