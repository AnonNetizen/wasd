# 任务模板：调数值（平衡迭代）

> **AI 修改说明**：修改本任务模板前先读 `docs/AI协作/文档维护指南.md`。模板变化通常需要同步 `docs/AI协作/README.md`、`docs/AI协作/上下文预算.md`、相关 subagent / command（如有）、`AGENTS.md` 的高频任务表（如入口变化）。

## 必读
1. 当前平台编码规则入口（见 `AGENTS.md` 第 3 步）第 3 节（数据与逻辑分离）
2. `client/data/README.md`（数值配置手册）
3. 目标 `client/data/*.json` 的目标条目
4. （改后）`tests/replays/golden_*.replay` 的存在情况

## 核心约束
- ✅ 只改 `client/data/` 下文件
- ❌ 绝不改代码常量
- ✅ 新增 / 改名字段时同步 `client/data/README.md`
- ✅ 大改动后**跑黄金回放**确认回归影响是有意的

## 步骤
1. 找到目标条目 → 改值
2. 确认字段单位 / 范围已在 `client/data/README.md` 写清
3. 跑 pre-commit（schema 校验 + 字段范围）
4. 大改动（如基础属性 / 难度曲线）：用 `Replay.play(golden_*)` 跑黄金回放，对比 diff
5. 如果是有意改动，更新黄金回放（重新录一组）+ 在 commit message 注明影响

## 自检
- [ ] 没碰代码
- [ ] 字段值在合法范围
- [ ] 字段说明文档仍准确
- [ ] hook 全过
- [ ] 大改动有回放回归证据

## 不需要做的
- 一般不需 ADR（除非是难度曲线哲学级改动）
- 一般不需 AI 记忆同步
