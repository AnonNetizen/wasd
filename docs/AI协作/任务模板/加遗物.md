# 任务模板：加一个遗物 / 道具

## 必读（按顺序）
1. `.codebuddy/rules/game-coding-rules.md` 第 6 节（道具/遗物系统数据驱动）
2. `docs/词表与契约.md` 第 1~3 节（stat / effect / event 白名单）
3. `client/data/relics.json` 黄金样例条目
4. `client/templates/relic_template.json`（结构参照）

## 核心约束
- ❌ 不写代码、不加分支
- ✅ 只新增一条 JSON 数据
- ✅ 所有 `stat` / `effect` / `event` id 必须**已在 `词表与契约.md` 登记**

## 步骤
1. 想清楚遗物想要的效果 → 拆成 `modifiers`（数值类）和 `behaviors`（行为类）
2. 检查每个 `stat` / `effect` / `event` id 是否在词表 → 不在就**先**走「加效果原语」任务模板登记并实现
3. 在 `client/locale/strings.csv` 加 `name_key` 与 `desc_key`（占位符 `{value}` 用于动态数值）
4. 在 `client/data/relics.json` 末尾追加一条，结构对齐黄金样例
5. 跑 pre-commit（或 `validate_contract.py`）—— 失败按报错改

## 自检
- [ ] 没有写任何 `.gd` 代码
- [ ] 所有 id 在词表
- [ ] locale key 都已添加
- [ ] 通过 DataLoader 校验（hook 全过）

## 不需要做的
- 不更新 ADR（除非引入了新原语，那是另一个任务）
- 不更新 GDD / 项目记忆（数据条目级变更不算重大决策）
