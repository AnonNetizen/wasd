# 任务模板：加一个遗物 / 道具

> **AI 修改说明**：修改本任务模板前先读 `docs/AI协作/文档维护指南.md`。模板变化通常需要同步 `docs/AI协作/README.md`、`docs/AI协作/上下文预算.md`、相关 subagent / command（如有）、`AGENTS.md` 的高频任务表（如入口变化）。

## 必读（按顺序）
1. 当前平台编码规则入口（见 `AGENTS.md` 第 3 步）第 6 节（道具/遗物系统数据驱动）
2. `docs/词表与契约.md` 第 1~3 节（stat / effect / event 白名单）；若是破限内容，额外读第 12 节（capability / tag）
3. `client/data/README.md`（数据字段、modifiers / behaviors 格式）
4. `client/locale/README.md`（name / desc 文案 key 与占位符规则）
5. `client/data/relics.json` 黄金样例条目
6. `client/templates/relic_template.json`（结构参照）

## 核心约束
- ❌ 不写代码、不加分支
- ✅ 只新增一条 JSON 数据
- ✅ 所有 `stat` / `effect` / `event` id 必须**已在 `词表与契约.md` 登记**
- ✅ 文案只填 `name_key` / `desc_key`，译文进 `client/locale/strings.csv`
- ✅ 突破默认限制的遗物 / 道具必须声明 `tag_limit_break` 与对应 capability，不能靠 `if relic_id == ...` 特判

## 步骤
1. 想清楚遗物想要的效果 → 拆成 `modifiers`（数值类）和 `behaviors`（行为类）
2. 检查每个 `stat` / `effect` / `event` id 是否在词表 → 不在就**先**走「加效果原语」任务模板登记并实现
3. 如果效果突破默认规则（栏位、开火、瞄准、移动、摄像机等），检查 `capability` / `tag_limit_break` 是否在词表第 12 节登记；没有就先登记并走设计评审
4. 在 `client/locale/strings.csv` 加 `name_key` 与 `desc_key`（占位符 `{value}` 用于动态数值，规则见 `client/locale/README.md`）
5. 在 `client/data/relics.json` 末尾追加一条，结构对齐黄金样例
6. 跑 `python tools/sync_contracts.py --check` 与 `python tools/validate_data.py`（或 pre-commit）—— 失败按报错改

## 自检
- [ ] 没有写任何 `.gd` 代码
- [ ] 所有 id 在词表
- [ ] 破限内容已声明 capability / tag，且没有按 id 特判
- [ ] locale key 都已添加
- [ ] `zh_CN` / `en` 已补齐，所有语言占位符一致
- [ ] 通过 DataLoader 校验（hook 全过）

## 不需要做的
- 不更新 ADR（除非引入了新原语，那是另一个任务）
- 不更新 GDD / 项目记忆（数据条目级变更不算重大决策）
