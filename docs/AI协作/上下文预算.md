# 上下文预算（按任务读取文件）

> 目的：避免 AI agent 拿到任务就盲目全仓搜索，降低 token 浪费、提升一致性。
> 原则：**只读必要文件，按白名单精确取**。任何任务超出预算前先反思是否走偏。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是任务读取范围权威；新增任务类型或改预算时，必须同步对应任务模板、`docs/AI协作/README.md`、`AGENTS.md`（如进入高频任务）、`docs/AI记忆/项目记忆.md`。

---

## 通用前置（任意任务都先读）

1. `docs/AI协作/快速开工.md` —— 低 token 热路径
2. `docs/AI记忆/current_state.json` —— 当前阶段、候选下一步、最近热验证
3. 当前平台编码规则入口（见 `AGENTS.md` 第 3 步）—— 强制规则（每次都读）
4. `docs/AI导航.md` 相关段落；通常先看第 4 节扩展点速查，必要时再读系统依赖图
5. `docs/AI记忆/项目记忆.md` 是冷存储；只有需要长期背景、ADR 摘要或历史脉络时再按相关节读取

> 通用前置目标控制在低 token 热路径内；完整长期记忆、GDD、ADR、测试策略和知识库索引按任务触发读取。

> `draft/` / `DRAFT/` 是用户人工草稿禁区，不属于任何默认上下文预算；除非用户明确点名授权处理该目录，禁止读取、搜索、修改、整理或引用。遵守该规则是默认行为，不需要在每次回复中主动声明。

## 任务复杂度分级（先判档，再取文件）

AI 接到任务后，先用下表判断复杂度，再按下一节“按任务的精确预算”选择具体文件。复杂度分级只决定流程深度，不替代任务类型预算。

| 档位 | 典型任务 | 上下文与流程 | 验证 / 收尾 |
|------|----------|--------------|-------------|
| S 小任务 | 查状态、解释代码、改错字、单句文案、只读诊断、轻量命令输出 | 只读目标文件、必要规则和直接引用；不写长计划；不扩展到无关模块 | 轻量验证或说明未验证；通常不自动 commit；不更新长期记忆 |
| M 中任务 | 新增 / 修改单条数据、局部 UI 调整、局部 bugfix、单模块内部行为修正 | 读对应任务模板或目标模块文档、目标源码 / 数据、相关 README；必要时读词表 / 测试策略相关段 | 跑对应 lint / data / smoke；按影响同步局部文档；通常不做正式 review，除非行为风险较高 |
| L 大任务 | 跨多文件功能、运行时行为变更、数据 schema、存档 / 回放 / UI 栈、工具链或阶段工作包推进 | 读当前工作包、目标模块文档、GDD / ADR 相关段、测试策略和目标源码；先给本次切片计划，再实现 | 跑完整对应验证；同步权威文档、AI 记忆和会话日志；大型代码改动提交前做事实型 review；默认自动 commit |
| XL 方向 / 规则任务 | 改核心玩法路线、推翻或新增 ADR、调整 AI 开工 / 上下文预算 / 平台入口、引入外部工具链 | 先评估价值、性价比、复杂度、风险和替代方案；确认权威来源与联动文件后再动手；必要时先澄清授权边界 | 必须记录 ADR 或长期文档决策；同步索引 / AI 记忆 / current_state；运行 docs health 和相关 JSON 校验；是否 commit 按变更规模判断 |

降档原则：如果任务可以低风险、可撤销地在更小档位完成，不要为了“完整”读取更多文档。升档原则：一旦发现影响跨模块契约、玩家可见行为、存档 / 回放 / 数据 schema、AI 开工流程或长期决策，立即升到 L / XL 并补读对应权威文件。

## 外部 AI 工具仓库读取策略

对 ECC 这类 agents / commands / skills / hooks / MCP 混合的大仓，默认按以下顺序控制上下文：

1. 先读 README、quick reference、LICENSE 和 package metadata，记录来源快照。
2. 用目录清单盘点所有工具面：agents、commands、skills、rules、hooks、scripts / CLI、MCP configs、platform adapters、plugins、examples、contexts。
3. 先分类，再全文读。与 Godot/GDScript、项目 AI 协作、验证、安全、上下文预算直接相关的候选才全文打开。
4. 最多 3 轮补充检索。每一轮都要说明新增信息改变了什么结论；没有改变就停止。
5. 在“研究结束 / 执行开始”边界沉淀清单，避免后续 agent 反复重扫同一大仓。
6. 外部 hooks、MCP、CLI、dashboard、plugin、vendor tree 默认不安装；需要启用时必须转入明确授权和单独评估。

ECC 的当前全量吸收记录见 `docs/AI协作/ECC工具吸收清单.md`。

## 越界提醒

读完上述清单仍解不了任务？**停下来反思**：

1. 任务是否真的能在数据/单文件层面解决？（规则 6：新道具 = 新数据，不是新分支）
2. 是不是缺少效果原语？应当先在词表登记。
3. 是不是真的要碰逻辑层？那需要先在 `决策记录.md` 加 ADR。

**不要为了"理解全局"而扫整仓**——那等于把 AI 协作的成本拉到人工水平。

## 元规则

- 每次任务结束，AI 应在 `AI记忆/会话日志/YYYY-MM-DD.md` 一句话回顾"实际读了什么文件"，超预算 30% 以上要写明原因。
- 任何反复出现的"超预算任务"，应当转化为一个**新任务模板**收录到 `任务模板/`。
- 跨阶段正式项目任务优先转化为**工作包**收录到 `docs/AI协作/工作包/`，避免每次重读完整规划 / GDD / 导航。
- `current_state.json.last_verified` 只保留最近热路径验证；完整验证历史写入当日会话日志，避免机器状态文件长期膨胀。
