# 测试策略

> 本文档收口项目所有测试相关约定，作为**唯一的测试方法权威**。
> 配套：CICD 路线图（`docs/CICD规划.md`）、编码规则（见 `AGENTS.md` 第 3 步的当前平台规则入口）、回放设计（`游戏设计文档.md` 9.9）、平衡测试（9.10）。
>
> **核心理念**：测试是**多层冗余**的，不是单点。每层抓不同 bug，缺哪一层都会出事。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是测试义务权威；改测试层级 / 覆盖率 / 里程碑矩阵时，常见联动为 `docs/CICD规划.md`、`docs/AI协作/实时验证回路.md`、规则自检清单、`docs/决策记录.md`、`docs/AI记忆/项目记忆.md`。

## 1. 五层测试金字塔（总览）

```
                  ▲
                 ╱ ╲
                ╱手 ╲       L5 手动回归 / 探索性测试（发版前）
               ╱──── ╲
              ╱平衡  Sim╲     L4 平衡 sim（批量 AI 玩家，统计平衡性）
             ╱──────── ╲
            ╱黄金回放回归 ╲   L3 整局回归（确定性回放 vs 当前代码）
           ╱──────────── ╲
          ╱  Headless 启动 ╲ L2 烟雾测试（项目能起来）
         ╱────────────────── ╲
        ╱ GUT 单元测试        ╲ L1 单元测试（核心模块逻辑）
       ╱────────────────────── ╲
      ╱  静态校验 + 契约校验      ╲ L0 编译前（pre-commit + CI 阶段 1）
     ╱──────────────────────────── ╲
    ──────────────────────────────────
```

| 层 | 名称 | 工具 | 反馈延迟 | 抓什么 bug |
|----|------|------|---------|----------|
| L0 | 静态/契约校验 | pre-commit hook + CI | **秒** | 裸字符串、未登记 id、schema 错误、死链、版本不同步 |
| L1 | 单元测试 | GUT | 秒~分钟 | 核心算法/边界/状态机正确性 |
| L2 | Headless 启动 | `godot --headless --quit` | 分钟 | 脚本编译错、缺 autoload、场景引用断链 |
| L3 | 黄金回放回归 | `Replay.play(golden_*)` + diff | 分钟 | 数值漂移、行为意外改变、确定性破坏 |
| L4 | 平衡 sim | `tools/sim.gd --runs N` | 小时 | build 强度失衡、过强/过弱组合、胜率异常 |
| L5 | 手动回归 / 探索 | 人工按 checklist | 数十分钟 | UI 体验、手感、未预料场景 |

> 每层缺一不可。L0~L2 是日常守门，L3~L4 是迭代质量，L5 是发版门槛。

## 2. 各层详细约定

### 2.1 L0：静态 / 契约校验
**实施**：见 `docs/CICD规划.md` 阶段 1（A 文档 / 数据校验 / B 词表契约 / D commitlint）+ `docs/AI协作/实时验证回路.md`。

**必查**：
- 文档死链 / `修改建议.md` 编号唯一 / `决策记录.md` ADR 连续 / 版本号同步
- 词表白名单（stat / effect / event / setting / locale / action / pool_id / damage_type / status_effect / audio_id / rng_stream）—— 当前由 `tools/sync_contracts.py --check` 与 `tools/validate_data.py` 守门，任何未登记 id 即 fail
- JSON / 数据 schema 校验（当前覆盖 `player.json`、`meta_progression.json`、MVP `mvp_config.json`，后续扩到 `relics.json` 等所有数据）
- Conventional Commits 格式
- GDScript lint + format（代码落地后）
- 强制类型化（无 untyped 函数签名进 main）
- **代码常量未手改**（`client/scripts/contracts/*.gd` 自动生成，与 md 一致）

**反馈方式**：fail-fast，输出**「文件 + 字段路径 + 期望值」**，让 AI/人据此自我纠错。

### 2.2 L1：GUT 单元测试
**实施**：见 `docs/CICD规划.md` 2.H。

**必测清单（按里程碑见 §3）**：

#### 横向基础设施（autoload，M1 必测）
| 模块 | 必测点 |
|------|-------|
| `Localization` | `tr()` 缺 key 行为；占位符替换；语言切换刷新 |
| `Settings` | 默认值；改值即时广播；持久化 + 加载；越界拒绝 |
| `Analytics` | `track_event` 字段集中；裸事件名拒绝；落盘格式 |
| `RNG` | **同主 seed 各子流序列稳定**；`weighted_pick` 分布；`luck_bias` 偏移 |
| `GameClock` | 暂停冻结；`time_scale` 影响；假时钟注入 |
| `Replay` | 录制完整性；同 seed 重放确定性；版本指纹一致 |
| `GameState` | 状态机非法跳转拒绝；信号顺序；与 `paused` 联动 |
| `UIManager` | push/pop 栈正确性；模态联动暂停；多层焦点 |
| `PoolManager` | acquire/release 计数；预热；溢出告警；`_pool_reset` 调用 |
| `SaveManager` | version/kind/slot/data_hash 头字段；原子写入 + `.bak`；迁移链跑通；损坏文件备份；`meta` 与 `run` roundtrip；与 `Settings` 不冲突 |
| `AudioManager` | id 路由 Bus；max_polyphony 节流；暂停 ducking |

#### 业务核心（M3+ 必测）
| 模块 | 必测点 |
|------|-------|
| `ModifierEngine` | `(基础+加法)×乘法` 公式；多源叠加；移除幂等 |
| `Combat` | 单入口路径；类型抗性；暴击判定走 `RNG.combat`；`cannot_kill` 标志 |
| `StatusEffectComponent` | 5 种 stack_rule 全覆盖；过期清理；与 ModifierEngine 联动 |
| `DataLoader` | 黄金样例必通过；未登记 id 必失败；fail-fast 输出格式 |
| `Spawner` | 同 seed 同分布；视野外回收 |
| `MetaProgressionSystem` | 结算奖励计算；购买成本与等级上限；解锁条件；永久修正器注入；`SaveManager` roundtrip |

**覆盖率目标**：
- 横向 autoload + Combat + ModifierEngine：**≥80% 行覆盖**
- 其他业务模块：**≥60%**
- 数据/UI 场景：不强求

**禁忌**：
- 测试本身不得使用裸 `randi()` / `Time` —— 用 `RNG.set_run_seed` + 假时钟
- 不测内部私有字段 —— 通过公开接口验证

### 2.3 L2：Headless 启动验证
**实施**：见 `docs/CICD规划.md` 2.G。

**做什么**：每个 PR 跑 `godot --headless --quit`，2 分钟内能干净退出 = 通过。
**抓什么**：脚本语法/类型错、缺 autoload 注册、`preload` 路径错、循环依赖、场景节点路径断链。
**升级版**：未来可加 `tools/smoke.gd`，跑通 1 分钟最小循环（开局 → 移动 5 秒 → 主动结束）。

### 2.4 L3：黄金回放回归
**实施**：见 GDD 9.9 + ADR #16 + CICD 4.M。

**流程**：
1. 在关键里程碑录一组 golden replay（玩家或 AIPlayer 跑 30~60 秒）
2. 存到 `tests/replays/golden_<scenario>.replay`，进 git LFS
3. CI 跑 `tools/replay_runner.gd <golden_files>`：
   - 用录制时的 seed + 输入序列重放
   - 对比帧状态（玩家位置、敌人血量、子弹位置、score）
   - 任意 diff > 容差（默认 0）→ fail，输出**首个 diff 帧 + 字段**

**何时更新黄金样例**：
- 改了数值/原语**有意改变行为** → 重录一组（commit message 注明影响）
- 修 bug 不应改变行为 → 黄金样例不动

**黄金样例必备组**（最小集，落地时录）：
| 文件 | 场景 | 时长 |
|------|------|------|
| `golden_basic_run.replay` | 出生 → 击杀 10 敌人 → 升级一次 | 60s |
| `golden_pause_resume.replay` | 玩 5s → 暂停 5s → 继续 5s | 15s |
| `golden_full_death.replay` | 完整死亡流程到结算 | ~30s |
| `golden_relic_synergy.replay` | 拾取 2 个有协同的遗物，验证修正叠加 | 60s |

### 2.5 L4：平衡 Sim
**实施**：见 GDD 9.10 + ADR #17 + CICD 4.N。

**何时启用**：M7（核心循环跑通后）。
**做什么**：headless 跑 1000 局 × 不同 seed，输出 CSV：
- 存活时长分布、击杀数、最终等级
- 每个遗物的"被选率 / 持有时胜率"
- 过强组合 top-10、无人选遗物清单
**怎么用**：每周/每次大改后跑一遍，diff 上一次的报告。

### 2.6 L5：手动回归 / 探索性测试
**何时**：每个 milestone 末尾 + 发版前。
**Checklist** 见 §5。

## 4. 性能预算（与 L4 联动）

性能不达标视同测试失败。预算先粗后细，落地后用 sim + Profiler 校准。

| 指标 | 目标（PC，1080p） | 触底警戒 |
|------|------------------|---------|
| 平均 FPS | ≥ 60 | < 50 时拉警 |
| 99% 帧时间 | ≤ 20 ms | > 33 ms 即 fail |
| 子弹峰值数（屏幕内） | ≤ 500 | > 800 必查池溢出 |
| 敌人峰值数（屏幕内） | ≤ 200 | > 300 |
| 内存（启动 5 分钟） | ≤ 512 MB | > 800 MB |
| 首帧到可玩 | ≤ 2 s | > 5 s |
| 数据热重载（一次） | ≤ 100 ms | — |

**度量方式**：
- 开发期：Godot Profiler + `PoolManager.stats()` 每帧水位
- CI：headless `tools/perf_probe.gd` 跑 60 秒采样 → 报表
- 触底警戒触发时：CI 标黄但不阻塞 PR；连续 3 次或跌破 fail 线 → 必须修

## 5. 手动回归 Checklist（发版前）

按顺序过一遍。任何一项 fail 都不能发版。

### 5.1 启动 / 主菜单
- [ ] 冷启动 ≤ 2s 进主菜单
- [ ] 设置菜单可打开，所有项可改可保存
- [ ] 切换语言（zh_CN ↔ en）UI 文本即时刷新，无残留旧语言
- [ ] 退出游戏不卡死，无错误日志

### 5.2 核心循环
- [ ] 开局后玩家居中，背景滚动，无相机抖动
- [ ] WASD 与手柄左摇杆 8 方向移动顺畅，无延迟
- [ ] 方向键、手柄 D-pad、手柄右摇杆瞄准 4 方向，自动开火按 `fire_rate`
- [ ] 子弹命中敌人造成伤害，飘字、命中音效、击杀粒子全有
- [ ] 经验掉落、自动吸附在 `pickup_range` 内
- [ ] 升级触发选择面板；默认 3 选 1，固定高 `luck` / 固定 seed 可验证概率 4 选 1；选择后**即时**生效
- [ ] 死亡触发结算面板，可重开

### 5.3 暂停 / UI 栈
- [ ] Esc / 手柄 Start 或 Menu 暂停，敌人/子弹/计时全停，菜单仍可点
- [ ] 暂停中再次触发 `pause` action 继续，状态完全恢复
- [ ] 设置菜单可在暂停中打开（双层 UI 栈），关闭后还原暂停状态
- [ ] 暂停菜单“保存并退出”会写入 `run` 存档并回主菜单，不产生局外结算奖励
- [ ] 升级选择期间游戏暂停，无操作残留；候选数量、候选 id 与选择可进入回放记录

### 5.4 数据驱动
- [ ] 改 `client/data/relics.json` 的某个数值，重启或热重载后即时生效
- [ ] 给 `relics.json` 写一个未登记 id → DataLoader fail-fast，错误信息含文件名+字段+期望值
- [ ] 删一个本地化 key → UI 显示原 key 而非崩溃

### 5.5 存档
- [ ] 元进度存档退出后重启依然在
- [ ] 局内暂停保存退出后，主菜单出现“继续游戏”，加载后恢复玩家、敌人、子弹、掉落、经验、遗物、RNG 与 GameClock 状态
- [ ] 升级选择 / 暂停态保存退出后，加载能回到正确 UI 恢复点
- [ ] 一局死亡、结算或放弃后，`run` 存档被删除或标记 consumed，不能重复结算刷局外奖励
- [ ] 存档写入使用临时文件替换；模拟半写入时可回退 `.bak`
- [ ] 手动改存档 `version` 字段为旧值 → 自动跑迁移
- [ ] 手动破坏存档 JSON → 备份到 `user://saves/.broken/` 且提示重置

### 5.5-A 局外成长
- [ ] 结算后按 `meta_progression.json` 获得局外货币与账号经验，单局上限生效
- [ ] 购买永久升级后余额扣除、等级提升、UI 文案刷新，重启后仍保留
- [ ] 永久升级通过 `ModifierEngine` 影响下一局属性，不直接改 `player.json`
- [ ] 达到账号等级或购买节点后授予对应 unlock id，未满足条件的节点不可购买
- [ ] 新建存档默认解锁项与 `default_unlocked` 一致

### 5.6 录制回放
- [ ] 一局结束后 `user://replays/` 出现 `.replay` 文件
- [ ] 重放该文件能完整复现该局（位置/血量/分数 100% 一致）
- [ ] 黄金回放在当前代码下全过

### 5.7 性能
- [ ] 玩 5 分钟，FPS 稳定 ≥60，无明显卡顿
- [ ] 5 分钟后内存 ≤ 512 MB
- [ ] 大量子弹场景（弹幕高峰）不卡顿

### 5.8 设置
- [ ] 重绑定键盘 / 手柄输入并保存，重启后依然生效
- [ ] 关闭"数据收集"后无埋点写入
- [ ] 关闭"自动录制回放"后无 `.replay` 写入
- [ ] 全屏/窗口/分辨率切换正常
- [ ] 三个音量条独立可调

### 5.9 异常
- [ ] 运行时切焦点（Alt+Tab）→ 按 `gameplay.pause_on_focus_loss` 行为
- [ ] 拔网（如果有联网功能）不崩
- [ ] 运行时插拔或关闭硬件输入设备（手柄等）不崩，回到键盘输入仍正常

## 6. 测试代码组织

```
client/
  tests/
    unit/                  # GUT 单测
      test_modifier_engine.gd
      test_rng.gd
      test_save_manager.gd
      ...
    integration/           # 集成（多 autoload 联动）
      test_pause_flow.gd
      test_save_load_roundtrip.gd
      ...
    replays/               # L3 黄金回放
      golden_basic_run.replay
      golden_pause_resume.replay
      ...
  tools/
    sim.gd                 # L4 平衡 sim 入口
    replay_runner.gd       # L3 回放对照器
    perf_probe.gd          # 性能采样
    smoke.gd               # L2 烟雾测试（最小循环）
```

**命名约定**：
- 单测文件 `test_<被测模块>.gd`
- 测试函数 `test_<行为>_<期望>`（如 `test_apply_damage_with_armor_reduces_amount`）
- 用 GUT 的 `before_each` / `after_each` 重置 `RNG` 与假 `GameClock`，避免测试间污染

## 7. AI agent 改完的测试义务

**任何代码改动**（数据除外）按下表选定测试责任：

| 改动类型 | 必跑 | 必新增 / 更新 |
|---------|------|--------------|
| 加效果原语 | L0 + L1 | 新增对应 GUT 测试 + 黄金回放（如影响平衡） |
| 加遗物/敌人/道具数据 | L0 | 数据条目跑 schema + 词表校验即可，不需要单测 |
| 加角色 / 破限 capability 数据 | L0 | schema + 词表 §12 校验；若只改数据不需要单测 |
| 加局外成长数据（货币 / 升级 / 解锁） | L0 | schema + 词表 §13 校验；若只改数据不需要单测 |
| 新增破限 primitive / strategy | L0 + L1 + L3 | 新增单测；如改变整局行为必须补 / 更新黄金回放 |
| 改 autoload 接口 | L0 + L1 + L2 | 单测必须更新 + headless 启动必须过 |
| 改伤害公式 / 状态效果叠加 | L0 + L1 + L3 | 单测 + 黄金回放（很可能要重录，commit 注明） |
| 加 UI 弹窗 | L0 + L2 + L5（暂停 checklist） | UIManager 集成测加一例 |
| 改存档 schema | L0 + L1 | 必须有迁移 + 迁移测试 |
| 改 `run` 续局快照字段 / 暂停保存流程 | L0 + L1 + L2 + L5（存档 checklist） | SaveManager roundtrip + 暂停保存/加载集成测；影响确定性时补黄金回放 |
| 改局外成长逻辑 / 购买规则 | L0 + L1 | MetaProgressionSystem 单测；若影响开局属性需补黄金回放 |
| 改性能敏感模块 | + L4 性能采样 | 与上轮 perf_probe 对比 |

**违反测试义务等同未完成**——参见编码规则自检清单。
