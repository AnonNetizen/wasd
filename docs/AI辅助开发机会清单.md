# AI 辅助开发机会清单

> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。
> 本文档的权威范围：记录“游戏运行时不接 LLM，只利用 AI 辅助写代码 / 数据 / 工具”时可参考的玩法、内容生产管线和开发工具机会。
> 常见联动：新增或采纳方向时同步 `docs/功能建议池.md`、`docs/TODO.md`、`docs/AI导航.md`、`docs/AI知识库索引.md`、`docs/AI记忆/current_state.json` 与 `docs/AI记忆/项目记忆.md`；若某项被用户明确点名落地，再补 GDD / ADR / 工作包 / 模块文档 / 测试策略。

## 0. 定位

本文档是给后续讨论用的“机会清单”，不是已采纳路线图，也不是实现授权。

核心前提：

- 游戏运行时不接 LLM，不依赖联网生成，不把不可控 AI 放进核心玩法。
- AI 的价值放在开发期：写代码、写数据、写验证工具、写编辑器、写批量内容草案。
- 游戏内最终表现仍然是离线、确定性、可回放、可校验的数据驱动系统。
- 人类负责方向取舍和品味，AI 负责放大产能，工具负责防止内容乱来。

适配本项目的默认边界：

- 可调内容进 `client/data/`，玩家可见文本进 `client/locale/strings.csv`。
- 新 id 先登记 `docs/词表与契约.md`，再跑 `tools/sync_contracts.py`。
- 随机走 `RNG`，时间走 `GameClock`，伤害走 `Combat`，状态走 `StatusEffectComponent`。
- UI 走 `UIManager`，高频实体走 `PoolManager`，存档走 `SaveManager`。
- 不为某个角色 / 遗物 / 技能 / 模式写一次性 `if id == ...` 分支。

## 1. 总体设计结论

AI 写代码变强后，最值得利用的是：让独立项目承担以前人力很难维护的“内容生产管线”。

以前难做的东西：

- 大量遗物和技能协同。
- 多种对玩家敌人行为。
- 有主题的地图 / 机关 / 事件组合。
- 自动化平衡模拟和回放差异诊断。
- 数据编辑器、内容质量 lint、曲线预览工具。
- 可复用 DSL / primitive / behavior 积木。

现在可以做的方式：

- 先实现少量稳定 primitive。
- 让 AI 批量生成数据候选和工具代码。
- 用 schema、lint、smoke、golden replay、perf probe 把关。
- 人类只挑最有趣的一批进入正式内容。

不建议的方式：

- 不建议运行时 LLM 生成剧情、关卡或敌人逻辑。
- 不建议做 AI NPC 对话作为早期核心卖点。
- 不建议让 AI 直接生成不可校验的脚本逻辑。
- 不建议跳过词表 / schema / replay，把“生成得快”变成“坏得快”。

## 2. 玩法与内容机会

### AID-01 怪物种间生态（已退役）

**状态**：ADR #144 已删除敌人种间猎食、逃跑、目标选择、互相伤害和导演相关元数据；本方向不再是候选机会，后续 AI 不得从本清单自动恢复。若未来重新讨论，必须由用户明确提出并建立新 ADR。

退役原因：

- 模块 JSON 已负责敌人组合和路线内容。
- 种间扫描、关系字段和互伤增加运行时、schema 与测试成本，却没有稳定形成玩家可读收益。
- 当前 EnemyAI 只维护对玩家的追击、快速近战、冲锋、守家、环绕和远程行为；中心分离仅防重叠。

### AID-02 遗物协同从加数值升级到流派包

**一句话**：用 AI 帮忙批量设计围绕一个主题的遗物、技能、状态、敌人和验证清单。

候选流派：

- 燃烧流：点燃、扩散、燃烧死亡爆炸、对燃烧目标暴击。
- 过载流：技能消耗更高，但释放后短时间极强。
- 拾荒流：经验球、尸体、机关残骸都能转成资源。
- 巢蚀流：越靠近危险区越强，但持续掉血或被标记。
- 召回流：子弹飞出后回旋，配合穿透 / 分裂形成弹幕回收。
- 低血狂战流：血越低越强，但治疗受限。

落地方式：

- 先补可复用 effect / status / behavior primitive。
- 用 `relics.json`、`skills.json`、`growth_pools.json` 组合流派。
- 不按单个 relic id 写逻辑。

### AID-03 巢变异 / 战区主题规则

**一句话**：让每局有一个可读主题，而不是只有刷怪数值变化。

候选主题：

- 燃烬巢：火焰机关更多，burn 相关奖励权重提高。
- 猎场巢：掠食者和猎物生态增强。
- 静默巢：周期性沉默主动技能，逼玩家靠走位和武器。
- 丰饶巢：经验更多，但清道夫怪也会抢经验。
- 裂隙巢：出现临时传送裂隙，玩家和敌人都能穿越。

落地方式：

- 作为 `game_modes.json` 或挑战配置的轻量覆盖。
- 影响资源池、spawn wave、hazard 权重、地图兴趣点。
- 所有随机仍走固定 seed 和 `RNG` 子流，保证回放可复现。

### AID-04 地图 / 机关编排工具

**一句话**：AI 帮开发工具生成兴趣点和机关组合，人类微调，PCG 只负责可复现摆放。

候选兴趣点：

- 精英守卫的遗物节点。
- 火焰机关走廊，可引怪通过。
- 安全补给区，但周围布置高速近战与远程交叉压力。
- 高风险高收益的巢核碎片区。
- 机关与敌人波次组合成小型战术题。

落地方式：

- 扩展 `map_layouts.json` 的 manual / pattern 配置。
- 建地图节点图生成器或导出预览。
- 先服务 Demo 手工打磨，不急做完全随机。

### AID-05 技能 / 状态效果组合网络

**一句话**：围绕轻量 GAS、ability tag 和 StatusEffect，做可组合状态关系。

候选组合：

- 燃烧 + 风刃 = 扩散火星。
- 沉默目标后，对其造成额外物理伤害。
- 标记目标后，技能优先锁定它。
- 中毒敌人死亡时留下毒池。
- 冻结敌人被火焰击中时爆裂。

落地方式：

- 状态、叠加规则、ability tag 都先入词表。
- 组合效果表达为可复用 primitive。
- L1 smoke 覆盖暂停冻结、续局恢复和 tag 生命周期。

### AID-06 死因复盘 + 下一局建议

**一句话**：死亡结算不仅显示输赢，还解释玩家为什么输，以及下一局可尝试什么。

候选输出：

- 你主要死于接触伤害。
- 你在 4 分钟后清怪效率不足。
- 你多次选择防御，但缺少范围输出。
- 建议尝试燃烧 / 穿透 / 拾取范围流派。

落地方式：

- 不需要 LLM，先用 Analytics / replay summary + 规则模板。
- 文案走 locale，判断规则走数据。
- 先做 5~8 条高置信建议，避免胡说。

### AID-07 构筑图鉴 / 协同发现系统

**一句话**：把玩家学习 build 的过程做成可解锁、可提示、可追踪的长期目标。

候选功能：

- 燃烧类遗物已收集 2/4。
- 已拥有点燃来源，推荐寻找扩散或爆炸。
- 某遗物与高射速武器相性更好。
- 隐藏协同：燃烧 + 分裂 + 穿透。

落地方式：

- 先做开发期协同矩阵，再决定游戏内显示多少。
- 图鉴状态走 `SaveManager.meta`。
- 提示不能泄露过多，保留发现感。

### AID-08 每日 / 每周巢情报挑战

**一句话**：固定 seed + 固定规则组合 + 本地最好成绩，形成复玩理由。

候选包装：

- 今日巢情报：燃烬巢。
- 敌方特征：火焰机关密集，猎物怪增殖。
- 推荐策略：高机动 / 冰冻 / 范围清理。
- 首次存活 8 分钟解锁图鉴条目。

落地方式：

- 短期只做本地每日，不做在线排行榜。
- 日期 / 时区要明确，避免回放不一致。
- 后续可接 `PlatformServices` / 成就 / 统计。

### AID-09 数据化战区导演

**一句话**：根据玩家状态调整下一波压力类型，但规则必须可解释、可回放。

候选规则：

- 玩家输出太强，增加高速猎杀怪。
- 玩家移动太慢，减少封路机关，增加远程压力。
- 玩家长时间没拿到技能资源，刷高风险资源点。
- 玩家一直绕边界，刷边界巡逻怪或兴趣点。

落地方式：

- 作为数据化 Director rule，不用黑箱 AI。
- 规则输入只用可记录状态，随机走 `RNG.spawn` / `RNG.world`。
- replay 记录触发原因，便于调试。

### AID-10 Build 试炼房 / 训练关

**一句话**：把流派教学和爽点验证做成短关卡。

候选试炼：

- 燃烧试炼：用火焰扩散清连锁敌群。
- 穿透试炼：站位让子弹穿过多目标。
- 机关试炼：引怪穿过 FEA-12 脉冲机关。
- 技能试炼：在沉默窗口外释放主动技能。

落地方式：

- 用专用 game mode / map layout 实现。
- 每个试炼可作为内容包附属验证。
- 先做开发用，再决定是否放进游戏内。

### AID-11 敌巢进化日志

**一句话**：让敌巢看起来会记录玩家表现并产生可见变化。

候选变化：

- 你依赖燃烧，下一局火抗单位出现概率上升。
- 你击杀大量猎物怪，掠食者数量下降。
- 你频繁依赖过载弹流，敌方部署压制射速或逼迫换位的节点。

落地方式：

- 不做无限惩罚，做轻量、可预测、可选择的变异。
- 变化进入 `meta` 或 challenge 配置。
- 包装上贴合《破巢者》的“敌巢活体战区”。

### AID-12 AI 辅助本地 Mod / 内容编辑器

**一句话**：把项目的数据驱动架构变成未来社区内容优势。

候选功能：

- 生成一个火焰主题遗物数据草案。
- 生成一个猎物型敌人 profile。
- 生成一个 8 分钟挑战模式。
- 检查本地 mod 数据是否符合 schema。

落地方式：

- 游戏内仍只加载 `ModLoader` 允许的数据 patch。
- AI 可以作为外部工具或编辑器助手，不进入游戏运行时。
- 不执行玩家脚本，不扩展核心契约。

### AID-13 战斗高光摘要

**一句话**：一局结束后给玩家可分享的高光摘要。

候选摘要：

- 最大连杀。
- 最危险的 10 秒。
- 最高 DPS 时刻。
- 最强遗物组合。
- 06:42 靠机关连杀 37 个敌人。

落地方式：

- 从 replay / Analytics summary 计算，不依赖 LLM。
- 文案模板走 locale。
- 可作为结算面板和截图传播点。

### AID-14 巢核事件链

**一句话**：给生存刷怪局加入明确行动目标。

候选链路：

- 发现通道锚点。
- 击破护盾节点。
- 守住采样器 30 秒。
- 抢夺巢核遗物。
- 选择撤离或继续深入。

落地方式：

- 作为数据化 objective / encounter chain。
- 地图兴趣点和奖励系统成熟后再做。
- 先一条短链，不做长剧情。

### AID-15 个性化难度标签

**一句话**：用规则标签替代简单 Easy / Normal / Hard。

候选标签：

- 清怪压力低，走位压力高。
- 机关密集，敌人较少。
- 资源稀缺，成长更慢。
- 高爆发，低容错。
- 生态混战，怪物互斗更多。

落地方式：

- 标签映射到 game mode 规则和资源池覆盖。
- UI 展示要清楚，不要让玩家猜。
- 不做隐藏动态难度，避免信任问题。

## 3. 开发工具与内容生产管线机会

### AID-16 遗物协同矩阵工具

**一句话**：自动扫描遗物、技能、状态、标签和效果，生成协同矩阵。

用途：

- 找到孤岛遗物。
- 找到过强组合。
- 生成图鉴提示。
- 帮 AI 定向补“某流派缺防御件 / 后期件 / 触发件”。

可能产物：

- `tools/analyze_synergy.py`
- `docs/AI协作/报告/synergy_matrix.md`
- 游戏内图鉴的数据源草案。

### AID-17 Build 原型生成器

**一句话**：输入一个流派目标，生成候选数据包和验证 checklist。

示例输入：

- 高射速低伤害毒流。
- 靠机关杀敌的拾荒流。
- 技能冷却循环流。
- 低血高收益狂战流。

输出内容：

- 遗物草案。
- 技能 / 状态 / 敌人建议。
- 需要的新 primitive 列表。
- 手动试玩 checklist。
- 需要跑的 smoke / replay。

### AID-18 Boss / 精英攻击模式 DSL

**一句话**：用数据脚本描述 Boss / 精英阶段，不为每个 Boss 手写行为。

示例概念：

```text
phase 1:
  every 3s fire ring
  every 5s dash to player
  below 60% hp summon prey swarm
phase 2:
  add rotating laser
  spawn hazard nodes
```

落地方式：

- 设计 `boss_patterns.json` 或类似 DSL。
- 写 parser / validator / runtime interpreter。
- 首片只支持少量动作：环形弹、冲锋、召唤、机关节点。

### AID-19 机关组合语法

**一句话**：把单个机关扩展成可复用 encounter pattern。

候选组合：

- 脉冲塔 + 推力风场。
- 火焰喷口 + 易燃地块。
- 毒池 + 减速带。
- 激光门 + 定时开合。
- 召唤巢 + 防护节点。

落地方式：

- 先做开发期 pattern 生成器。
- 再决定是否进入 `map_layouts.json` 正式 schema。
- runtime 仍复用 `HazardSystem` / `PoolManager` / `Combat`。

### AID-20 敌人行为积木库

**一句话**：把只面向玩家或环境目标的 EnemyAI action / scorer 做成可组合积木。

候选积木：

- 接近玩家。
- 保护目标。
- 吃掉落物。
- 召唤小怪。
- 蓄力冲锋。
- 保持距离射击。
- 低血回守出生点。
- 死亡分裂。

落地方式：

- 新行为先做通用 action，不写 enemy id 分支。
- profile 只配置权重、条件和参数；玩家是唯一战斗目标。
- 每个 action 至少有一个 smoke 或固定场景断言。

### AID-21 地图兴趣点生成器

**一句话**：用节点图生成兴趣点布局，而不是纯随机撒点。

节点类型：

- 宝箱点。
- 精英点。
- 机关试炼点。
- 商店 / 补给点。
- 巢核事件点。
- 高风险资源点。

落地方式：

- 工具输出 `map_layouts.json` 草案或可视化报告。
- 人工微调后入库。
- 与有限地图 bounds 和 safe radius 联动。

### AID-22 局内任务目标系统

**一句话**：为每局加入短目标，避免只绕圈等时间。

候选目标：

- 90 秒内击破巢瘤。
- 引 20 个敌人死于机关。
- 守住信标 30 秒。
- 击杀掠食者精英。
- 收集 3 个巢核碎片。

落地方式：

- 做 objective primitive：计数、计时、防守、击杀、收集。
- 奖励走已有成长 / 遗物 / 资源系统。
- UI 先做轻量 HUD 文本，按英文验收。

### AID-23 自动平衡模拟器

**一句话**：用简单 AIPlayer 跑大量局，输出平衡报告。

报告指标：

- 平均存活时间。
- 各敌人击杀贡献。
- 各遗物选择率 / 胜率。
- 哪些流派过强。
- 哪些升级选项没人选。
- 哪些时间段死亡率异常。

落地方式：

- 先用简单贪心 AIPlayer，不追求像人。
- 输出 CSV / JSON / Markdown。
- 与 replay runner 形成平衡护栏；`perf-probe` 仅在用户明确要求性能测试时运行。

### AID-24 回放差异可视化

**一句话**：把 golden replay mismatch 变成可读报告。

报告内容：

- 从哪一帧开始不同。
- 玩家 HP 差异。
- 敌人数量差异。
- RNG 子流调用差异。
- 伤害事件差异。
- 升级候选差异。

落地方式：

- 扩展 `replay-runner` 输出 diff artifact。
- 优先 Markdown / JSON，不急做 UI。
- 让 AI / 人能快速判断是预期数据指纹变化还是 bug。

### AID-25 自动生成试玩 checklist

**一句话**：根据改动类型生成手动试玩清单。

示例：

- 新遗物：测拾取、叠加、描述、英文显示、存档恢复。
- 新敌人：测出生、AI、碰撞、死亡、经验掉落、回放。
- 新技能：测冷却、资源、目标、暂停、状态、续局。

落地方式：

- 可先做文档模板或脚本。
- 与 `docs/测试策略.md` §7 对齐。
- 作为工作包收尾项。

### AID-26 内容质量 lint

**一句话**：在字段合法之外，检查内容设计质量。

候选规则：

- 描述是否提到真实效果。
- 数值是否明显超范围。
- 同稀有度强度是否离谱。
- 是否有未使用 tag。
- 是否有遗物没有任何协同。
- 是否有敌人 profile 过于相似。

落地方式：

- 初期作为 advisory，不阻塞 CI。
- 高置信规则再升级为 lint gate。
- 输出带文件 / id / 建议原因。

### AID-27 装备 Mod 成长预览工具

**一句话**：把 Gear Mod 掉落、升级成本和容量约束变成可检查的成长图。

检查内容：

- Mod 来源与掉落概率。
- 成本曲线。
- rank 效果与 drain 变化。
- 总成本。
- 第一小时可获得路线。
- 是否存在过贵、无掉落来源或永远装不下的 Mod。

落地方式：

- 先导出 Markdown / Mermaid。
- 后续可做图形预览。
- 与 `meta-smoke` 补充，不替代运行时验证。

### AID-28 技能资源系统扩展包

**一句话**：让不同英雄的主动技能资源真正不同。

候选资源：

- 怒气：受伤 / 近战击杀获得。
- 热量：连续释放升高，过热后锁技能。
- 弹药：地图拾取补充。
- 充能：站在机关区或击杀精英获得。
- 血祭：消耗生命释放强技能。

落地方式：

- 资源获取 / 消耗 / 上限 / 衰减都数据化。
- 技能只引用 resource id，不写死 mana。
- HUD 需要能显示多资源或当前角色资源。

### AID-29 动态难度曲线编辑器

**一句话**：把 `spawn_waves.csv`、敌人预算和机关压力导出成曲线。

曲线维度：

- 敌人密度。
- 总血量预算。
- 伤害压力。
- 精英出现频率。
- 机关压力。
- 经验 / 奖励投放。

落地方式：

- 先做静态报告。
- 后续接 sim 结果，显示“预期压力”和“实际死亡率”。
- 用于 F9 / Demo 打磨。

### AID-30 内容包开发流程

**一句话**：以后不零散加内容，而是按主题内容包推进。

一个内容包可包含：

- 一个主题巢。
- 3 个敌人。
- 1 个精英。
- 1 组机关。
- 8 个遗物。
- 2 个技能。
- 1 条挑战规则。
- 对应 smoke / replay / checklist。

落地方式：

- 做 `docs/AI协作/任务模板/加内容包.md` 或工作包模板。
- AI 生成候选，人类筛选。
- 每包有明确回归命令和手动验收。

## 4. 推荐优先级

### 最适合近期讨论的四项

| 方向 | 推荐理由 | 风险 |
|------|----------|------|
| AID-20 敌人行为积木库 | 直接提升敌人对玩家的战斗变化 | 需要保证 profile 数据和 runtime action 边界清楚 |
| AID-18 Boss / 精英攻击模式 DSL | 大幅提升精英 / Boss 内容产能 | DSL 一旦设计太大，验证成本会上升 |
| AID-16 遗物协同矩阵工具 | 让 build 深度可控扩张 | 初期只能做 advisory，不要假装能自动判断所有乐趣 |
| AID-23 自动平衡模拟器 | 让项目敢批量加内容 | AIPlayer 不像真人，报告只能辅助判断 |

### 最适合 Demo 体验的三项

| 方向 | 推荐理由 | 风险 |
|------|----------|------|
| AID-03 巢变异 | 每局主题更明确，适合展示差异 | 资源池和波次改动会影响 golden / 平衡 |
| AID-14 巢核事件链 | 把“活得久”变成“有目标的破巢行动” | 需要地图兴趣点和奖励系统支撑 |
| AID-06 死因复盘 | 提升失败后的下一局动机 | 判断规则要保守，避免误导玩家 |

### 最适合长期内容生产的三项

| 方向 | 推荐理由 | 风险 |
|------|----------|------|
| AID-17 Build 原型生成器 | 用 AI 快速产出候选包 | 必须人工筛选，不可直接入库 |
| AID-26 内容质量 lint | 内容越多越能防漂移 | 误报较多时只能 advisory |
| AID-30 内容包开发流程 | 把 AI 产能转成稳定交付节奏 | 短期玩家感知弱 |

## 5. 点名推进时的推荐说法

可以直接这样点名：

- “评估 AID-20，先只做设计，不实现。”
- “推进 AID-16 第一片，做工具报告，不改游戏内 UI。”
- “把 AID-03 拆成一个 F10 工作包。”
- “比较 AID-18 和 AID-09，告诉我哪个更适合 Demo。”
- “围绕燃烧流做一个 AID-30 内容包草案。”
- “用 AID-25 给这次技能改动生成手动试玩 checklist。”

如果用户只说“继续利用 AI 代码能力做点东西”，AI 应先让用户选择方向，或基于 `current_state.json` 说明候选，不应从本文档自行挑一项实现。
