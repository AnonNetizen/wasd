# 局内刷取玩法参考研究

> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。
> 本文档的权威范围：记录本项目 2D / 固定斜俯视射击局内刷取、撤离结算、兴趣点和装备 Mod 循环可参考的外部游戏。
> 常见联动：若从本文档采纳新机制，需要同步 `docs/游戏设计文档.md`、`docs/AI协作/工作包/F12-ShortLootRuns.md`、`docs/AI导航.md`、必要时 `docs/决策记录.md` 与对应模块文档。

## 1. 定位

本文档是参考研究，不是已采纳路线图。它回答一个问题：我们的默认模式已经转向 8-12 分钟短刷图、兴趣点、小巢核、撤离结算和 Gear Mod / dust 回报，那么同类或相邻游戏里有哪些设计值得拆解。

项目当前目标不是做纯幸存者类，也不是硬核拟真撤离射击。更贴切的参考组合是：

```text
SYNTHETIK 的枪感与武器构筑
+ Escape From Duckov / ZERO Sievert 的带回式刷取
+ HELLDIVERS 1 的目标完成后撤离压力
+ Enter the Gungeon / Nuclear Throne 的 2D 射击可读性
+ Hades / Nova Drift / 20 Minutes Till Dawn 的局内构筑语言
```

适配本项目时应继续遵守既定边界：默认标准模式不恢复局内经验 3 选 1，不做 25 分钟默认长局；奖励先进入 `run.pending_loot`，撤离成功才写入 `meta.gear_mods`；射击核心是移动、瞄准和按住 `fire` action，而不是自动攻击替玩家完成战斗。

## 2. 结论

第一优先级参考：

| 游戏 | 为什么优先 | 本项目对应问题 |
|------|------------|----------------|
| Escape From Duckov | 单机 PvE、俯视、搜刮、基地 / 装备成长、撤离带回，和 F12 的收益带回规则最近 | `pending_loot` 情绪价值、撤离前贪心、局外整理 |
| SYNTHETIK: Legion Rising | 俯视射击手感、武器差异、Android upgrades / build options 强 | 按住开火后如何仍有枪械个性和构筑深度 |
| HELLDIVERS Dive Harder Edition | 任务目标清晰，做完目标后进入撤离防守压力 | 小巢核击破后撤离区、尾声软加压 |
| ZERO Sievert | 俯视 extraction shooter，重视路线、风险、战利品与撤离 | 有限开放战区、兴趣点路线、死亡丢失 |

## 3. 分维度拆解

### 3.1 枪感与射击身份

重点参考 `SYNTHETIK`、`Enter the Gungeon`、`Nuclear Throne`。

本项目已经从自动攻击转向按住 `fire` action 持续开火，所以枪感必须承担“玩家在操作”的存在感。可学的是武器之间的性格差异，而不是硬核模拟细节。

适合迁移：

- 武器有明确节奏：高射速低伤、慢速高穿透、散射、爆裂、连发、蓄热、弹速差异。
- 命中反馈足：命中闪烁、击退、破甲、暴击、目标硬直、掉落飞出。
- 构筑围绕枪线服务：射速、弹速、穿透、分裂、连锁、弱点、命中触发、过热冷却。
- 精英 / 小巢核能逼玩家移动换角度，而不是原地压住开火。

不宜照搬：

- 复杂手动换弹、卡壳、弹匣管理、弹药经济过重。
- 过高准度惩罚和拟真后坐力，可能破坏短刷图爽感。
- 让技能或自动召唤抢走主武器表达。

落地建议：

- Gear Mod 优先补武器侧 Mod：`damage` 之外增加 `fire_rate`、`bullet_speed`、`pierce`、`split`、`on_hit_status` 等可组合方向。
- 每个武器族至少有一个“手感关键词”：稳定、扫射、爆发、穿透、散弹、过载。
- 兴趣点目标和精英守卫应该测试武器构筑差异：缓存箱守卫偏清杂，小巢核偏持续输出，精英巢点偏走位与点杀。

### 3.2 带回式刷取与撤离压力

重点参考 `Escape From Duckov`、`ZERO Sievert`、`HELLDIVERS`、`Quasimorph`。

本项目的核心不是“活满倒计时”，而是把收益拿到手以后是否安全带回。ADR #122/#123 已经确立：Gear Mod / dust 先进入 `run.pending_loot`，撤离成功才提交。

适合迁移：

- 进图前有轻量目标预期：这局想刷 dust、Mod、精英掉落或小巢核保底。
- 地图上有可见收益点：资源缓存低风险、Mod 缓存中风险、精英巢点高风险、小巢核是结算钥匙。
- 撤离不是按钮，而是一段最后压力：进入撤离区、守住短读条、敌群软加压。
- 死亡失败要展示丢失清单，让玩家理解“刚才差一点带回什么”。

不宜照搬：

- 背包格子管理和材料垃圾海。
- 过强的“死了亏一小时”焦虑。
- PvP extraction 的阴人、蹲撤离和长时间搜刮空窗。

落地建议：

- 结果面板继续强化“带回 / 丢失”的对比，尤其展示 Gear Mod 名称、稀有度和 dust 数。
- HUD 可以增加简洁的本局暂存包：`pending dust`、`pending mods`、`撤离后带回`。
- 小巢核击破后不应立刻安静，建议给 20-40 秒撤离压力窗口，由数据控制软加压强度。
- 保存退出只保存 run，不结算；这个边界应在 UI 文案里尽量清晰。

### 3.3 兴趣点路线与地图结构

重点参考 `ZERO Sievert`、`Hades`、`HELLDIVERS`、`Deep Rock Galactic: Survivor`。

本项目不做房间制迷宫，也不做空旷平原。F12 的地图应该是有限开放战区，玩家能快速读懂“去哪、值不值、能不能撤”。

适合迁移：

- 从投放点看见或很快发现 2-3 个收益方向。
- 每个兴趣点有一眼能懂的收益标签：资源、Mod、精英、小巢核。
- 路线选择要有机会成本：先拿安全资源提升容错，还是直接打高价值 Mod 缓存。
- 尾声让玩家考虑“再贪一个点，还是撤离”。

不宜照搬：

- Hades 的完整房间门路线，因为我们现在是开放战区。
- 大型随机迷宫，因为会稀释 8-12 分钟目标。
- 完全随机点位导致 Demo 不可控；关键点仍要允许数据手摆和 smoke 验证。

落地建议：

- 后续补小地图或方向标时，不需要复杂地图系统，先做兴趣点边缘指示器和距离提示。
- 资源缓存 / Mod 缓存应有功能色与 silhouette，和敌巢危险色分开。
- 精英巢点和小巢核要有 footprint，避免与普通陷阱混读。
- 地图结构优先保证 10 分钟内完成“2 个兴趣点 + 小巢核 + 撤离”。

### 3.4 局内构筑语言

重点参考 `20 Minutes Till Dawn`、`Nova Drift`、`Hades`、`Brotato`。

默认模式暂不启用局内经验升级 3 选 1，但这不代表局内没有构筑变化。我们可以把默认模式的局内变化更多压到掉落、临时武器强化、消耗品、缓存奖励和 Gear Mod 验证上。

适合迁移：

- 构筑标签清晰：子弹数量、穿透、状态、暴击、弹速、射程、命中触发。
- 强化有视觉反馈：弹道改变比面板数字更容易让玩家兴奋。
- 局外 Mod 和局内掉落能互相验证：这套 loadout 进图后明显改变战斗手感。
- 稀有掉落不一定每局都有，但每局应稳定带回某种可用进度。

不宜照搬：

- 默认模式每 30 秒弹一次升级选择，打断射击节奏。
- 构筑池过宽，导致首版数据平衡和回放验证成本爆炸。
- 只有数字 +5%，没有可感知变化。

落地建议：

- 默认模式先做“少量高辨识度局内奖励”：临时过载弹、短时穿透、缓存补给、精英掉落稀有 Gear Mod。
- 局内 3 选 1 可作为未来挑战 / 无尽 / 幸存者变体，通过 `growth_pools` 显式启用。
- Gear Mod 内容扩展时优先做 6-10 个能改变射击读法的首批 Mod，而不是大量微小数值 Mod。

### 3.5 敌潮与时间压力

重点参考 `Vampire Survivors`、`Brotato`、`Nuclear Throne`、`Deep Rock Galactic: Survivor`。

F12 的 8-12 分钟是目标窗口，不是硬倒计时。敌潮应该推动路线选择，而不是把玩家锁在原地清怪。

适合迁移：

- 时间窗分层：0-1 分钟热手，1-4 分钟第一收益点，4-7 分钟第二收益点，7-9 分钟小巢核，9+ 分钟软加压。
- 敌群类型服务路线：小怪压迫移动，精英守点，机关切割地形。
- 高压尾声让玩家更想撤离，而不是单纯堆怪到卡顿。

不宜照搬：

- 30 分钟长局强节奏。
- 屏幕上满是自动清掉的敌人，玩家只负责走位。
- 怪潮完全和兴趣点无关。

落地建议：

- 敌群应围绕兴趣点增援：Mod 缓存打开时刷小波守卫，精英巢点有守卫语义，小巢核有尾声压力。
- 9+ 分钟软加压可以优先加“追兵 / 远程压迫 / 机关节奏”，少用纯血量膨胀。
- `perf-probe` 要成为怪潮扩容的护栏，避免爽感来自无控制堆实体。

## 5. 对 F12 的直接建议

近期优先做：

1. **精英巢点守卫语义**：参考 `HELLDIVERS` 的目标压力和 `ZERO Sievert` 的危险区，让精英巢点不是一个孤立靶子，而是“守卫 + 机关 + 奖励”的小战斗。
2. **缓存箱奖励仪式**：参考 `Enter the Gungeon` 的宝箱反馈，缓存箱打开时要有短反馈、掉落飞出和 HUD 暂存提示。
3. **撤离尾声压力**：参考 `HELLDIVERS`，小巢核击破后撤离读条期间刷一小波追击，软加压但不硬封死。
4. **本局暂存包 HUD**：参考 extraction 游戏的带回压力，让玩家随时知道现在身上有什么值得保命。
5. **武器 Mod 内容首批扩展**：参考 `SYNTHETIK` 和 `Nova Drift`，优先做能改变弹道 / 命中反馈的 Mod。

暂缓：

1. 背包格子、材料垃圾海、基地建设深系统。
2. 默认模式局内 XP 三选一。
3. 25-30 分钟长局。
4. 多人合作目标、友伤、复杂指令输入。
5. PvP extraction 的埋伏 / 蹲撤离体验。

## 6. 设计检查清单

改 F12 或局内刷取内容前，可用下面的问题自检：

- 玩家进图后 30 秒内是否知道最近的收益方向？
- 3 分钟内是否发生第一次明确收益或选择？
- 玩家是否能说清“我现在身上的暂存战利品值不值得撤”？
- 小巢核是否是本局终点钥匙，而不是普通高血量障碍？
- 撤离是否有压力，但不是硬性失败倒计时？
- 奖励是否让下一局 loadout 或武器手感发生可感知变化？
- 敌潮是否在逼路线和空间，而不是单纯刷经验球？
- 失败结果是否清楚展示丢失了什么？
- 新机制是否仍走 `RNG`、`GameClock`、`Combat`、`SaveManager`、`GearModSystem` 等既定边界？
- 机制是否能通过 runtime smoke、gear-mod smoke、save smoke 或 golden replay 验证？
