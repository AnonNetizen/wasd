- 第一阶段新增内部 `module_authoring` EditorPlugin、`ModuleSceneBaker`、`module-bake` / 只读 `module-bake-check` / `module-bake-smoke`、`ModuleBakedData` 与旋转资源；相关路径才触发 pre-commit，authoring/plugin 从 release 排除。
- 一次性 Godot API 转换器已从 16 个注册 JSON 生成固定 `Ground` / `Obstacles` / `Decoration` / `Placements` 场景，并回烘焙 JSON/TRES；16 份 JSON 与迁移前逐字节一致，sealed 保持 candidate，其余审核状态未变。
- 第二阶段已把 `ModuleBakedData` 升到 schema v2：每个旋转预烘焙 16 种世界封边 obstacle pattern / 合并碰撞；`ModuleChunk` 删除 `_draw()`、碰撞节点创建和激活重算，`ModuleWorldManager` 场景预置九个 chunk 并按 3×3 活跃邻域复用。
- `GameplayRunLoop` 预置 manager / `SkillSystem`，Player / Enemy / SkillSystem 预置 `StatusEffectComponent`，HUD 预置 minimap；L1 与 module-world smoke 也改为实例化正式场景。注册表新增 `approved_source_hash`，即使 TRES 缺失也不能绕过内容变化后的 candidate 降级。
- 第二阶段 contracts、data/schema、三档 lint、semantic regression、export-tree、headless、bake、完整 / 技术切片 module-world、runtime、save、L1 与四条 golden runtime rerun 全过；map hash 保持 `63f49b0...a1083`，未运行 startup/perf probe。
- 第三阶段已把 Player / Enemy / Bullet / PickupOrb / HitSpark 的占位 `_draw()` 迁为场景内 `Polygon2D` / `Line2D` 子树；保留的地图范围、机关 footprint、撤离进度和 minimap 动态绘制均把样式暴露为场景导出属性，WorldBackground 不再每帧无条件重绘静态网格。
- HUD stats、升级候选、Gear Mod 列表与输入绑定改为实例化五个可编辑行模板；语义 lint 新增 `runtime-node-construction`，拦截正式 gameplay/UI 的 Node/Control `.new()` 后挂树，同时保留已登记对象池 factory 和 PackedScene 行模板边界。
- 第三阶段 headless、runtime、settings、gear-mod、L1 与 module-world smoke 已通过；最终全套检查和第三次提交待本轮收口，仍不运行 startup/perf probe。

## ADR #154 模块 JSON 可视化编辑与单向 TSCN 烘焙

- 用户指出直接把 `.tscn` 当人工 / AI 协作主源会引入脆弱文本、双向转换和所有权问题，因此 ADR #154 取代 ADR #153 的模块制作源部分；ADR #153 的 gameplay/UI 场景化继续有效。
- 16 个 `client/data/modules/*.json` 已迁为 schema v2：保留 terrain / placement，新增 Ground、Obstacles、Decoration 三层视觉声明；socket 只从边缘 floor 派生。`module_tile_catalog.json` 用稳定 `tile_id` 隔离 Godot source / atlas / alternative 细节；注册表改用 `approved_gameplay_hash`。
- 内部 `Module JSON` 已从侧栏 Dock 迁到与 `2D / 3D / Script` 同级的中央主编辑区；三栏界面只编辑 JSON，支持模块列表、11×11 画布、层 / 工具 / 四向预览、Undo/Redo、dirty、显式原子 Save、打开时磁盘 hash 冲突门禁、新建 / 复制以及 Validate / Bake / Approve。画布和属性页允许纵向收缩，不再限制 FileSystem / Debugger 停靠区分割器；首版没有重命名、删除、自动合并或 TSCN 反向导入。
- 初版 baker 只读取 JSON，支持全部或 `--module <id>`；当时 61 个允许旋转的场景生成到 `client/scenes/generated/modules/`，内含三层 TileMap、合并基础碰撞、四边封锁视觉 / 碰撞、玩法 / 表现 hash 与旋转 placement 快照。`module-bake-check` 在内存重建规范场景指纹，能识别缺失、过期或人工改动，不信任场景自报 metadata。

- 运行时在 assignment 建立 / 恢复时预加载唯一 `PackedScene`，九个 `ModuleChunk` 只挂载缓存实例，跨边缘最多替换三块；JSON 继续负责 assignment、导航、inactive placement 与 schema v1 等价 gameplay map hash，不在运行时构建 TileMap 或碰撞。
- 旧 `client/scenes/modules/*.tscn`、模块专用 `client/resources/modules/*.tres`、`ModuleBakedData` / `ModuleBakedRotation`、authoring root / marker 和 `--scene` / `--migrate-json` / TSCN→JSON 路径均已删除；release 包含生成场景并排除 Dock、editor script 和 bake runner。
- 迁移后的四条黄金运行时摘要保持既有 map hash `63f49b0cee98050b07f0feb68722894713a784a4b7238242506d077add0a1083`，没有重录 golden；全局 data fingerprint 因 schema v2 / 视觉数据按预期变化，回放使用显式 mismatch 许可只比较运行时摘要。未运行 startup/perf probe 或 Profiler。
- 中央主编辑区修正通过 `module-json-editor-smoke`、`module-bake-check`（61 个场景）、`module-bake-smoke`（87 条断言）、`module-world-smoke`、数据 / schema、三档 lint、headless boot、文档健康与 JSON 校验；semantic lint 无 advisory，未运行性能 probe。
- 随后按用户确认把方向重复生成物收敛为每模块唯一 `rotation_0.tscn`：baker schema 升至 v2，写入 bake 会删除其他 `rotation_*.tscn`，无写入 check 会把遗留方向文件视为过期。生成场景从 61 份降为 16 份；allowed rotations 继续约束 assignment，但不再扩增生成物。
- `ModuleWorldManager` 改为按 module id 预加载规范场景，`ModuleChunk` 对根节点应用 0/90/180/270° 旋转与 1600 px 方形枢轴补偿，并把世界封边反映射到规范方向子树。JSON assignment、导航、placement、schema v1 等价 map hash 与 run v4 没有改变；`module-bake-check` 已验证 16 份指纹，`module-bake-smoke` 以 112 条断言覆盖遗留方向发现、四向根变换和封边反映射，完整 `module-world-smoke` 与 `headless-boot` 通过。未运行性能 probe。

## ADR #155 角色与敌人专属继承场景

- 用户确认角色和不同敌人至少应拆成可复用的专属 TSCN，并继续使用 Godot 场景继承。新增 `PlayerBase` / `EnemyBase`，以及 `character_default` 与 `enemy_chaser` / `enemy_swarm` / `enemy_stalker` / `enemy_bulwark` / `enemy_spitter` 的真实继承场景；一次性 editor-only 迁移入口未提交。
- `characters.json` 升 schema v2 并新增 `scene_path`；`enemies.csv` 新增 `scene_path` / `pool_prewarm`、删除 `visual_color`。场景路径允许不同内容 id 复用，但必须位于正式 actor 子目录、存在且不能指向基础场景。
- 五种敌人改为 `enemy_chaser` / `enemy_swarm` / `enemy_stalker` / `enemy_bulwark` / `enemy_spitter` 独立池，移除旧 `enemy_ranged`；预热为 `8 / 5 / 3 / 4 / 8`，合计仍为 28。
- `GameplayRunLoop` 从预置 `PlayerHost` 挂载默认或快照角色，开局按唯一 `scene_path` 缓存 `PackedScene`，并根据敌人数据动态注册 / 预热池。TSCN 管静态颜色、归一化轮廓与节点结构，生命、速度、伤害、碰撞半径和 AI profile 继续外置；`Enemy.configure()` 不覆盖场景表现。
- 新增 `actor-scene-smoke` 并扩展 data/schema、runtime、L1、模块世界、存档和黄金回放断言。四条黄金回放的地图 hash、敌人类型 / 数量 / 位置 / 血量和其他玩法摘要保持不变，仅更新数据指纹；`golden_basic_run` 的精确 `pool_stats` 按五个独立池更新。按用户边界未运行 startup/perf probe 或 Profiler。
