## Steamworks Slime Lab 残存 Smoke 与快照 MTU 修复

- 用户明确要求把上一轮留下的三项残留一起处理；新增 ADR #136。世界矩形实现保持扩展 viewport 内居中固定 `540×960` 设计区不变，`battle_smoke` 改为根据公开设计尺寸和当前 viewport 计算原点，再验证 `500×870` 战斗区与边距。
- `SteamLabSave` 改为显式写入 `ConfigFile.encode_to_text()`、flush 和 close；`record_survival_time()` 仅在写盘成功后返回 `true`，失败会回滚内存纪录。`battle_smoke` 的 settings / save 备份、删除与恢复也显式关闭句柄并校验错误，新增不可写路径回滚和保存后重载覆盖。
- `SteamLabNetworkSession` 保持上层 `snapshot_received(Dictionary)` 与快照字段不变；底层用 FastLZ 压缩后按最多 900 字节拆成 unreliable chunk RPC，携带 64 位递增序号、原始大小、分片下标和总数。客户端只重组最新完整序列，拒绝超过 1 MiB、异常分片数量 / 下标、元数据冲突、非法 payload、解压失败或非 Dictionary 数据，离开 / 建立会话会清理缓存。
- 新增只读 `snapshot_wire_stats()`，host smoke 校验原始 / 压缩大小、分片数和最大分片；Lobby marker 保持 `steamworks_slime_v1`，wire 不兼容版本从 `lab_version=1` 升为 `2`，旧版与新版互拒。
- 首轮连续回归又暴露主动道具拾取 smoke 的旧时序抖动：完整世界 tick 会让随机障碍 / 敌人在拾取判定前把玩家推开。该断言改为直接调用拾取判定，隔离验证生成 / 拾取 / 替换语义，不改玩法实现；随后 `battle_smoke` 从第 1 次重新连续运行 5 次全部零失败。
- 自动验证与事实型 review 结果在本节完成后补入 `current_state.json`；真实 Steam Lobby / 邀请 / 冷启动 / 战斗同步仍需发布前用两个账号和设备验证，内部 ENet 回归不能替代。

## Steamworks Slime Lab 单人 / 多人存活纪录拆分

- 用户确认最长存活时间只拆为单人和多人两类；多人合并本地同屏与 Steam Host / Client，各 Steam 设备按 host 权威 Game Over 时间本地落盘。旧版混合纪录无法可靠归类，迁移时清空。
- 新增 ADR #137。`SteamLabSave` 引入 `RecordCategory`，`save.cfg` 的 records schema 升为 v2，分别保存 `best_single_survival_seconds` / `best_multiplayer_survival_seconds`；检测到旧 `best_survival_seconds` 会以双零重写并删除旧键，未知更高版本拒绝加载且不覆盖。
- 主场景按 `PlayMode` 选择分类：SINGLE 写单人，COUCH / STEAM_HOST / STEAM_CLIENT 写多人，MENU 忽略。每个分类独立比较最长时间，只有完整写入、flush、close 成功才提交内存值；失败只回滚目标分类。
- RecordsPanel 改为同时显示 `单人最长存活 / Single Player Best` 和 `多人最长存活 / Multiplayer Best`，两行独立显示无记录或 `MM:SS`，并扩大面板以覆盖英文布局。
- `battle_smoke` 新增双零默认、旧档清空迁移、v2 schema / 旧键移除、未来版本拒绝、独立提高 / 低值 / 非法值、目标分类写盘回滚、保存重载、PlayMode 映射、单人 Game Over 和中英文双行 UI 覆盖；`local_couch_smoke` 使用独立临时存档验证同屏 Game Over 只更新多人纪录。
- 验证通过：`battle_smoke --disable-steam` 串行 5 次零失败、`local_couch_smoke`、`steam_config_smoke --disable-steam`、Lab headless boot、内部 ENet host/client 双端 `ALL PASS`；host 最大快照 payload 低于 900 字节，日志无 `above the MTU`。正式 `client/`、Steam wire 与现有导出产物未修改，真实 Steam 仍按双账号 / 设备发布门禁验收。
