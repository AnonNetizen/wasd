## 决策
- **ADR #46**：完整项目采用深局外成长；`meta_progression.json` 数据化局外货币、账号等级、永久升级轨道和内容解锁，玩家状态持久化走 `SaveManager` 的 `meta` kind。
- **ADR #47**：启用最小 Stage 1 CI；`.github/workflows/docs-check.yml` 只跑 JSON、CSV、文档健康检查和 whitespace diff，复杂检查后续再加。
- **ADR #48**：完整项目需要强玩家存档；`SaveManager` 必须同时支持 `meta` 局外成长长期档案与 `run` 局内暂停退出续局档案，并采用版本迁移、原子写入、备份回退和损坏隔离。
- **ADR #49**：新增 7 个创意 / 策略 subagents，覆盖玩法设计、数值、IP、文案、UI 美术、游戏美术和宣发策略。
- **ADR #50**：新增沟通审查与需求评估规则；问有没有问题 / 风险时不硬找问题，提需求后先评估项目落地前景、性价比、复杂度和主要风险。
- **ADR #51**：落地数据校验 / 契约同步 / 轻量 Godot Bridge，形成词表、数据、locale、MVP 场景与 headless 启动的可验证闭环。
- **ADR #52**：确立 AI Git 提交策略；大更改完成后 AI 默认自动 commit，细微改动不提交，提交前必须检查 status / diff / log、跑验证并只 stage 本次任务文件。
- **ADR #53**：安装项目级 OpenCode skills；把成熟 Agent Skills / MCP / marketplace 模式本地化为 7 个按需加载流程，并记录资源筛选清单。
