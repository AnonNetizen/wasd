## 决策
- **ADR #31（最终口径）**：`.codebuddy/` 与 `.codex/` 共享核心项目约束，但不要求文件内容、目录结构或工具字段完全一致；允许按不同 AI agent / 平台优化。pre-commit hook 与 CI `health-check` 应校验核心规则版本、关键能力清单与文档登记，而不是内容哈希。
- **ADR #32**：`AGENTS.md` 是所有 agent 通用入口；`CODEX.md` / `OPENCODE.md` 只做平台加载适配，不能放宽核心规则。
- **ADR #33**：`.opencode/` 成为 OpenCode 平台配置入口；OpenCode 使用 `opencode.json` 注册命令、加载 instruction，agents/commands/rules 与 `.codebuddy/` / `.codex/` 保持核心语义同步。
- **ADR #34**：默认玩法规则不是硬编码上限；破限角色 / 道具必须通过 capability、content tag、effect / behavior primitive 或 strategy 扩展，禁止按 id 写一次性分支。
- **ADR #35**：长期代码模块、公共 API、signal、数据 schema、依赖方向、扩展点或测试义务变化时，同步详细 `docs/代码/` 模块文档；不要求每个小 helper 单独成文档。
- **ADR #36**：AI 面向用户默认中文回复；仅在用户明确要求、引用代码 / API / 命令 / 日志原文、目标文件语言要求或对外发布文本需要时使用其他语言。
- **ADR #37**：`draft/` / `DRAFT/` 是用户人工草稿禁区；AI 默认不得读取、搜索、修改、整理、格式化、总结或引用，除非用户明确点名授权。
- **ADR #38**：`MinimumViableProduct/` 是 MVP 隔离实验区；MVP 文档与客户端代码都放在该目录内；初始玩法约束为玩家不移动、方向键四方向射击、敌人四方向刷新。
- **ADR #39**：完整项目正式支持手柄输入；默认左摇杆移动、右摇杆 / D-pad 4 方向瞄准、手柄 A / South 主动道具、Start / Menu 暂停，设备差异收口在 InputMap / Settings 绑定层。
- **ADR #40**：完整项目代码文档必须采用详细模块文档；AI 可按模板自动创建/维护，但不得用自动抽取的简短摘要替代。
- **ADR #41**：`docs/` 升级为 AI 知识库；新增人工索引、机器索引、术语表和文档健康检查脚本，方便 AI 检索、判断权威层级并发现文档漂移。
- **ADR #42**：AI 知识库升级为 v2；机器索引增加 owner / review / canonical / must-read 元数据，健康检查扩展链接、覆盖、`related_code`、ADR 矩阵与风险报告，并接入三平台 `/health-check`。
- **ADR #43**：完整项目采用经验与升级选择系统；击杀掉落经验，升级默认 3 选 1，`luck` 概率性提升为 4 选 1，升级选项池内容另列待决策。
- **ADR #44**：AI 记忆改为三层结构；`项目记忆.md` 只做长期索引，`current_state.json` 保存机器可读当前状态，`会话日志/` 保存每日细节。
- **ADR #45**：完整项目建立人工可维护的数值 / 文案配置手册；`client/data/README.md` 管数值字段、单位和调参流程，`client/locale/README.md` 管多语言文案、语言列、key 命名和占位符规则。
- **MVP-9（局部决策）**：MVP 用 `data/mvp_config.json` 集中核心可调数值，但暂不引入完整项目 `DataLoader`；原因是需要快速人工调参，同时保持 MVP 轻量。
