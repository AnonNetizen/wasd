## 关键决策

| 决策 | 理由 |
|------|------|
| 回放系统升格为**第四条**横向基础设施（与本地化/设置/埋点并列） | Roguelike 平衡迭代必需的回归安全网；确定性硬约束必须框架阶段立 |
| 平衡测试 MVP 前只锁接口形态，不实现 | 现在做空中楼阁不划算，但接口走偏后不可回头 |
| AI 协作工程升格为**第五条**横向基础设施 | 把"AI 怎么协作"工程化，对齐业界 Cursor/Aider/Continue 等的实时反馈与可复用模板 |
| 新建 `docs/AI协作/` 与 `docs/AI记忆/` 并列 | 记忆 = 状态；协作 = 方式；二者不混 |
| 系统依赖图用 Mermaid 写进 `AI导航.md` 5.2 | 对齐"AI 改前看影响范围"的最佳实践 |
| pre-commit hook 与 CI 复用同一脚本 | 单源真理，避免规则在两处漂移 |
| 调试用 action 走可重绑定但仅 debug build 启用 | 既保持"无硬编码"原则，又不污染发布版 |
