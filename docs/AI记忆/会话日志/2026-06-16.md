## 多模式资源复用原则

- 输入：用户提出后续可能会有多种游戏模式，角色、遗物等资源不要做得强耦合，应易于复用。
- 评估：该需求与既有“扩展优先 / 数据驱动 / 禁止按 id 写特殊分支”一致，价值高；主要风险是过早做重模式系统，因此本次只落设计约束和数据边界，不实现模式系统。
- 产出：新增 ADR #65；同步 GDD、AI 导航、数据手册、三平台规则入口、项目记忆与 `current_state.json`。
- 结论：资源本体默认模式无关；后续 `game_modes.json` 等模式配置只负责组合资源池、权重、条件、禁用列表和轻量覆盖，禁止为某个模式复制资源或写 `if mode_id == ...` 内容分支。
