- 评估：第三档适合进入 CI 日志与 health-check，但不应默认阻塞；特殊 id 分支、业务脚本绕过 autoload、缺类型签名、缺 `# Doc:` 与未知 contract 常量都能自动扫描，但 warning 需人工判断。
- 产出：新增 `tools/lint_semantic_rules.py` 与 `tools/test_semantic_rules_lint.py`，默认输出 advisory warning 并退出 0，`--strict` 可局部升级为失败；`.github/workflows/docs-check.yml` 以 `|| true` 接入非阻塞步骤，回归测试仍硬失败；三平台 `/health-check`、三平台规则、CICD / 测试策略 / 实时验证 / 文档健康 / AI 导航 / 知识库索引 / CONTRIBUTING 已同步；ADR #78 记录该定位。
- 结论：第三档已脚本化为非阻塞语义 lint，当前正式 client 运行无 advisory warning。
