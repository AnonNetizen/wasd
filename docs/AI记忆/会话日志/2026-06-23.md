## 摘要

- 用户先要求按 `AGENTS.md` 初始化对话，已完成快速开工热路径读取。
- 用户询问现有技能系统后，确认当前 `SkillSystem` 是可复用主动技能首片：`skills.json` 定义技能，角色通过 `starting_loadout.skill_ids` 引用，资源池来自 `skill_resources`，旋风斩经 `Combat` 造成 AOE 伤害。
- 用户要求吸收 Unreal GAS 思想后，新增 ADR #94：项目采用 Godot 适配版轻量 GAS，只吸收 ability / tag / cost / cooldown / targeting / effect 管线，不照搬 Unreal 网络预测、replication 或 Blueprint 类型系统。
- 用户问下一步后，继续推进 ADR #95：状态效果首片接入轻量 GAS，`StatusEffectComponent` 管理沉默状态授予 / 移除 `ability_tag_silenced`、按 `GameClock` 过期并进入 SkillSystem 快照。
- 用户同意继续后，推进 ADR #96：Player 与 Enemy 正式接入 `StatusEffectComponent`，实体状态效果、owned ability tags 和状态快照进入真实目标与对象池复用边界。
- 用户确认“先做 burn”后，推进 ADR #97：burn 作为首个真实 DoT 状态接入项目版轻量 GAS，经 `StatusEffectComponent` 计时并通过 `Combat.apply_damage()` 结算 `fire + is_dot` 伤害。
- 用户随后确认“先把 burn 变成可玩内容，不做 poison”，新增 `skill_ignite_slash` 点燃斩作为真实数据技能；默认主键仍释放旋风斩，点燃斩作为第二个已加载技能和模式池内容存在。
- 用户指出敌人能跑到地图外，这是有限地图边界漏接；已修复为敌人生成 / 续局恢复时注入 `MapManager.bounds()`，Enemy 普通移动、冲锋、中心分离和快照恢复后统一 clamp 到有限地图内。
- 用户讨论小服务器条件下能做哪些玩法；已新增 `docs/小服务器玩法备忘.md`，记录基础异步玩法与更有辨识度的敌巢进化、失败转敌人、星域污染图等参考方案，明确不是已采纳路线图。
