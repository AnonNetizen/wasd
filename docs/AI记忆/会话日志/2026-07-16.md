## Steamworks Lab Smoke 永久防复发

- 用户要求把此前反复出现的 smoke 编排、真实存档风险、成功误判、ENet 静态端口 / 缓存锁和 `steam_appid.txt` 误删问题全部固化解决，并保证同仓库后续 Codex、Claude、CodeBuddy、OpenCode 对话使用同一流程。
- 新增 ADR #138。`tools/steamworks_lab_toolchain.py` 增加 `smoke --suite all|boot|steam-config|local-couch|battle|enet`；`all` 固定按 boot → Steam 配置 → 同屏 → battle 默认五轮 → ENet fail-fast。Windows 优先同目录 console 可执行文件，每个进程使用独立临时用户环境，GDScript smoke 同时检查退出码、精确 `ALL PASS` 和致命日志。
- ENet runner 使用动态 UDP 端口，等待 host 精确 `READY` 后启动 client；client 使用不含 `addons`、`build`、`.godot` 的轻量临时项目副本，runner 统一负责超时、终止、日志清理和 MTU 警告拒绝。
- `SteamLabSettings` 与主场景支持注入配置路径；battle、local couch、ENet smoke 都在节点 `_ready()` 前注入专用 settings / save fixture，不再备份、删除或迁移玩家真实 `user://settings.cfg` / `save.cfg`。runner 会在运行前后逐字节保护这两项真实文件和源码 `steam_appid.txt`。
- 回归补齐未来 records schema 写保护、Steam Client `phase_received(GAME_OVER) → apply_phase → phase_changed → 本地多人纪录落盘` 完整链路、场景实际采用测试配置路径、统一 local couch 成功标志和 ENet `--net-smoke-port` / host `READY` 协议。
- `tools/test_steamworks_lab_toolchain.py` 与 pre-commit 固化 console 解析、隔离环境、精确成功 / 错误标志、超时终止、动态端口、临时副本范围和源码 `steam_appid.txt=4955670` 保护。源码 App ID 文件永久保留，只允许发布导出包排除。
- 四平台 `godot-test-diagnostics` skill、Lab README、测试策略、工具适配指南、AI 协作说明、AI 导航、知识库索引、CICD / 实时验证说明与项目记忆已同步；Steamworks Lab 后续不再手写 Godot、五轮 battle 或 PowerShell 双进程命令。
