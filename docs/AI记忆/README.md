# AI 记忆（跨会话/跨机器协作记忆）

本目录用于让 AI agent 在**新机器 clone 仓库后**或**新会话开始时**能"接得上"之前的对话与决策。日常低 token 开工入口见 `docs/AI协作/快速开工.md`。

> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档只说明记忆目录结构与使用方式；改记忆维护规则时应优先修改 `docs/AI记忆/项目记忆.md` 第 9 节、`docs/AI记忆/current_state.json`、规则 14-B / 22、`docs/决策记录.md`。

## 文件结构

```
docs/AI记忆/
├── README.md         # 本文件，简介
├── 项目记忆.md        # 长期冷存储：项目快照 + 决策摘要 + 文件入口
├── current_state.json # 机器可读当前状态：阶段 + 下一步 + 最近热验证
└── 会话日志/         # 按日期归档的对话详细摘要
    └── YYYY-MM-DD.md
```

## 使用方式

### 用户视角
- 在**新机器** / **新会话**开始时，提示 AI："先读 `AGENTS.md`、`docs/AI协作/快速开工.md` 和 `docs/AI记忆/current_state.json`"。
- 需要长期背景、ADR 摘要或历史脉络时，再让 AI 读取 `docs/AI记忆/项目记忆.md`。
- 想回顾某天对话细节：查 `会话日志/YYYY-MM-DD.md`。

### AI 视角（主动更新模式 a）
- **每次重要对话 / 决策结束后**，主动按 `项目记忆.md` 第 9 节「更新约定」更新本目录。
- `项目记忆.md` 保持精简（< 200 行），当前状态覆盖写入 `current_state.json`，细节和完整验证历史进会话日志。
- 不需要用户每次提醒，但**内容只记结论与文件指针**，不复述全文，控制 token 成本。

## 维护原则

- **fail-graceful**：找不到对应日期的会话日志不算错，主文件能自洽就行。
- **可回溯**：所有决策仍以 `docs/决策记录.md` 为权威源；本目录是"导航 + 上下文"，不是决策本身。
- **机器无关**：内容用相对路径，不带任何机器/用户名/时区敏感信息。
- **机器可读**：`current_state.json` 只保存当前事实和最近热路径验证，不保存历史；字段由 `tools/docs_health_check.py` 校验。

详细约定见 [`项目记忆.md`](项目记忆.md) 第 9 节。
