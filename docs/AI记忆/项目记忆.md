## 1. 项目快照

- **名称**：wasd
- **类型**：俯视角 Roguelike 弹幕生存（《以撒的结合》+《吸血鬼幸存者》风格）
- **引擎**：Godot 4.6.3 + GDScript（强制类型化）
- **当前状态**：**文档/框架设计阶段**，无代码
- **设计文档版本**：v1.4
- **仓库结构**：`docs/`（文档）、`client/`（Godot 客户端，占位）、`server/`（暂占位）
- **远端**：`https://github.com/AnonNetizen/wasd.git`
- **License**：MIT

## 2. 核心设计支柱（一眼回忆）

| 支柱 | 一句话 |
|------|-------|
| 数据驱动 | 所有可调数值进 `client/data/*.json`，零代码调参 + 热重载 |
| 遗物 = 数据 | `modifiers`（数值）+ `behaviors`（行为）描述，**严禁为每个遗物写硬编码分支** |
| **5 条横向基础设施** | `Localization` / `Settings` / `Analytics` / `Replay` / AI 协作工程；以及 `RNG`、`GameClock` 单一源 |
| 玩家恒居中 | 相机挂玩家下，**关 limit 与 drag margin** |
| 词表契约 | 所有约定字符串走 `docs/词表与契约.md` 白名单，**禁止裸字符串/编造 id** |
| AI 友好工程 | 项目索引、词表、数据校验、黄金样例、ADR、模板齐备；**任务模板 + 上下文预算 + 角色分工 + 实时验证回路** |
| 暂停 | `get_tree().paused` + 菜单节点 `process_mode=ALWAYS`，键走 InputMap action `pause` |
| 确定性 | 随机走 `RNG`、时间走 `GameClock`，禁止裸 `randi()` / `Time.get_ticks_msec()`（回放回归的硬约束） |

## 3. 既定决策（ADR 摘要，详见 `docs/决策记录.md`）

18 条已落地决策（按编号）：
1. 类型：俯视角 Roguelike 弹幕生存
2. 引擎：Godot 4.6.3 + GDScript
3. 玩家恒居屏幕中央，关 limit / drag margin
4. WASD 8 方向移动 + 方向键 4 方向瞄准 + 全自动开火
5. 遗物 modifiers + behaviors 数据驱动
6. 数值外置到 `res://data/`（JSON）+ 热重载
7. 框架级多语言本地化（`tr("key")`，禁硬文本）
8. 框架级设置系统（`Settings` autoload + 信号 + `user://settings.cfg`）
9. 框架级数据埋点（`Analytics` autoload + `track_event`）
10. 采纳 E~I 系列建议（数值格式/遗物架构/平衡/美术规范/元进度）
11. AI 友好工程（项目索引/词表/数据校验/黄金样例/类型化/模板/ADR/日志/README）
12. 元规则：新规则归档 + 文档与实现同步（规则 19/20）
13. 暂停功能：`get_tree().paused` + 菜单 `PROCESS_MODE_ALWAYS` + 可重绑定 `pause` action
14. 仓库三段结构（`docs/` + `client/` + `server/`）
15. AI 协作记忆体系（`docs/AI记忆/` 主索引 + 会话日志，AI 主动更新模式）
16. **录制回放系统**为第四条横向基础设施（GDD 9.9）：`Replay` autoload + 确定性硬约束（`RNG` / `GameClock`） + 黄金回放回归
17. **平衡测试接口**预留（GDD 9.10）：`AIPlayer` 与 seed 注入；MVP 前只锁形态
18. **AI 协作工程**为第五条横向基础设施（GDD 9.11 + 规则 14-A）：`docs/AI协作/`（任务模板 + 上下文预算 + 角色分工 + 引擎集成 + 实时验证回路） + 系统依赖图 + CI 健康度/回放回归/平衡 sim

## 9. 更新约定（重要）

**触发条件（满足任一即更新）**：
- 任何 ADR 新增 / 修改
- 任何编码规则新增 / 修改
- 任何"待决策项"被采纳或新增
- 任何重要文件结构调整
- 任何工具链变更（CI、git 配置、新模板）
- 重要里程碑达成

**更新动作**（AI 主动执行，无需用户提醒）：
1. 修订本文件第 1~5 节（项目快照 / 决策 / 待决策 / 工具链）以反映现状
2. 在第 6 节追加当日条目（按时间倒序）
3. 在 `会话日志/YYYY-MM-DD.md` 写当日详细摘要（输入/产出/决策）
4. 修订第 7 节"下一步候选"

**节制原则**：
- 只记**结论 + 关键参数 + 文件指针**，不复述完整对话
- 单次会话总更新量目标 < 1KB
- 历史细节归入会话日志，主索引保持精简（< 200 行最佳）

**不触发条件**（避免噪声）：
- 单纯的代码风格修复 / typo
- 不涉及决策的纯实现工作（在会话日志记一笔即可）
