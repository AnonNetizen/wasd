## 1. 项目快照

- **名称**：wasd
- **作者署名**：中文 `伦敦阿农`；英文 `Anon London`
- **类型**：俯视角 Roguelike 弹幕生存（《以撒的结合》+《吸血鬼幸存者》风格）
- **IP 方向**：ADR #88 已定为 **《破巢者》**（英文暂定 `Nestbreakers`），独立权威文档为 `docs/IP设定.md`：未知原因导致其他宇宙与本宇宙的通道突然打开，原本拥有银河系星际文明的人类被打散，只剩首都星域相对稳定；多英雄主动突入敌方“巢”，在怪潮中夺取遗物、升级构筑并尝试打穿巢核、切断通道或削弱敌方源头；“巢”是敌方核心据点 / 生产源头 / 通道锚点 / 意志中枢的泛称，不限定为虫巢。
- **引擎**：Godot 4.7 + GDScript（强制类型化；新写 / 修改脚本以 Godot 4.7 官方 GDScript style guide 为基础风格）
- **当前状态**：详见 `docs/AI记忆/current_state.json`；长期事实为 MVP 已由用户确认验证完成且隔离目录已移除，AI 工作框架已搭建完成，正式项目 F1~F8 当前验收均已完成，F9 第一轮 Demo 内容 / 表现打磨也已由用户确认收口。F9 后续已按用户点名新增 ADR #88《破巢者》IP 方向、ADR #89 数据驱动生态 EnemyAI 首片、ADR #90 可复用主动技能系统首片、ADR #91 多语言 UI 英文长度基准、ADR #92 详细数值 HUD 面板、ADR #93 有限地图 + 可人工调参 PCG / FEA-12 机关测试链路、ADR #94 项目版轻量 GAS 首片、ADR #95 状态效果 ability tag 生命周期首片、ADR #96 Player / Enemy 实体状态宿主，以及 ADR #97 burn DoT：`StatusEffectComponent` 可让 `skill_effect_apply_status` 施加 `silence` 或 `burn`，授予 `ability_tag_silenced`，随 `GameClock` 过期 / tick，并经 `Combat.apply_damage()` 结算 `fire + is_dot` 伤害，状态与 tick 计时进入 SkillSystem / Player / Enemy run 快照；随后新增 `skill_ignite_slash` 作为真实数据技能，默认主键仍释放旋风斩，点燃斩作为第二个已加载技能和模式池内容施加 burn DoT；敌人有限地图移动边界已补齐，生成 / 续局恢复时由 `GameplayRunLoop` 注入 `MapManager.bounds()`，Enemy 普通移动、冲锋、中心分离和快照恢复后都会 clamp 到 bounds。`docs/功能建议池.md` 与 `docs/AI辅助开发机会清单.md` 都只是可选参考菜单，不自动授权 AI 自行实现；后续工作以用户最后明确指令为准。

## 2. 核心设计支柱（一眼回忆）

| 支柱 | 一句话 |
|------|-------|
| 数据驱动 | 所有可调数值进 `client/data/`，平表数值优先 CSV、复杂配置优先 JSON；字段说明进 `client/data/README.md`，零代码调参 + 热重载 |
| 双语本地化 | 玩家可见文本进 `client/locale/strings.csv`，当前首批维护 `zh_CN` / `en`；AI 自动补另一语言首版译文，人工最终复核；UI 尺寸以英文 `en` 文案长度作为设计和验收基准 |
| HUD 详细数值 | 按住 `show_stats_panel` action 显示详细数值面板、松开隐藏，不暂停、不入 UI 栈；默认 Tab，可通过 Settings 重绑定 |
| 深局外成长 | `meta_progression.json` 数据化局外货币、账号等级、永久升级和解锁，状态走 `SaveManager` 的 `meta` kind |
| 强玩家存档 | `SaveManager` 同时支持 `meta` 局外成长和 `run` 暂停退出续局，要求迁移、原子写入、备份回退和损坏隔离 |
| 遗物 = 数据 | `modifiers`（数值）+ `behaviors`（行为）描述，**严禁为每个遗物写硬编码分支** |
| 技能 = 轻量 GAS 数据 | `skills.json` 描述 ability tags、activation 条件、成本、目标、效果和冷却；角色 / 道具 / 模式只引用 skill id，资源池可扩展；当前内置旋风斩和点燃斩，伤害技能走 `Combat`，持续状态走 Player / Enemy / SkillSystem 自身的 `StatusEffectComponent`，DoT 用 `damage_type` / `magnitude` / `tick_interval` 并经 `Combat` tick |
| 敌人生态 = 数据 | 敌人基础数值在 `enemies.csv`，生态行为在 `enemy_ai_profiles.json`；用 content tag + action profile 表达猎物、掠食者、领地等关系，禁止按 enemy id 写 AI 分支 |
| 有限地图 + 可调 PCG | 地图不再无限扩展；`map_layouts.json` 声明 bounds、出生点、安全半径、刷怪边距、PCG 机关规则和人工摆点，`MapManager` 负责可复现生成 / 保存，并约束玩家 / 敌人中心移动不离开 bounds |
| 扩展优先 | 默认玩法规则不是硬编码上限；破限角色 / 道具必须用 capability、tag、primitive 或可复用 strategy 声明式扩展 |
| 模式友好资源复用 | 角色、遗物、道具、敌人、成长奖励等资源本体默认模式无关；游戏模式只通过资源池、权重、条件、禁用列表和轻量覆盖组合资源 |
| 本地 Mod 接口 | 玩家 mod 先作为 `user://mods/<mod_id>/mod.json` 数据包进入，走 `ModLoader` + `DataLoader` 声明式 patch；创意工坊未来只负责分发，不执行玩家脚本 |
| 平台服务接口 | Steam API 优先通过 `PlatformServices` 预留：成就、统计、状态显示、overlay、Lobby / 邀请和用户身份统一进门面；其他平台后续走 provider adapter |
| 未来多人友好边界 | 当前只做单人，不实现网络；输入、伤害、阵营、回放、存档和埋点预留 actor / participant / team / intent 概念 |
| **完整 autoload 矩阵** | 3 协作（`Localization`/`Settings`/`Analytics`）+ 2 确定性（`RNG`/`GameClock`）+ 1 回放（`Replay`）+ 1 平台服务（`PlatformServices`）+ 3 协调（`GameState`/`UIManager`/`PoolManager`）+ 2 资源（`SaveManager`/`AudioManager`）+ AI 协作工程（`docs/AI协作/`） |
| 玩家恒居中 | 相机挂玩家下，**关 limit 与 drag margin** |
| 词表契约 | 所有约定字符串走 `docs/词表与契约.md` 白名单，**禁止裸字符串/编造 id** |
| 代码-文档同步 | 长期代码模块 / 公共 API / 数据 schema / 依赖变化必须同步详细的 `docs/代码/` 模块文档 |
| AI 知识库 | `docs/AI知识库索引.md` + `docs/_kb_index.json` + `docs/术语表.md` + `tools/docs_health_check.py` |

## 9. 更新约定（重要）

**触发条件（满足任一即更新）**：
- 任何 ADR 新增 / 修改
- 任何编码规则新增 / 修改
- 任何"待决策项"被采纳或新增
- 任何重要文件结构调整
- 任何工具链变更（CI、git 配置、新模板）
- 重要里程碑达成

**更新动作**（AI 主动执行，无需用户提醒）：
1. 修订本文件第 1~5 节（项目快照 / 决策 / 待决策 / 工具链）以反映长期事实
2. 更新 `docs/AI记忆/current_state.json`（当前阶段、最新 ADR、待决策项、下一步、最近验证）
3. 在第 6 节保留 / 覆盖当日一行摘要（**一行表格条目 + 日志链接**，不展开细节）
4. 在 `会话日志/YYYY-MM-DD.md` 写当日详细摘要（输入/产出/决策/验证）

**节制原则**：
- 只记**结论 + 关键参数 + 文件指针**，不复述完整对话
- 单次会话总更新量目标 < 1KB
- 历史细节归入会话日志，短期状态归入 `current_state.json`，长期索引保持精简（< 200 行最佳）

**自动瘦身（无需用户提醒）**：
- 每次更新本文件前先检查总行数；**临近 200 行立即瘦身**——不等用户问。
- 第 6 节单日条目超过 1 行 → 折叠为「日期 + 一句话摘要 + 日志链接」格式。
- 7 天前的条目 → 压缩为「YYYY-MM-W（周）：一句话」周聚合。
- 30 天前的条目 → 整体折叠为「月份 + 该月主要里程碑」月聚合。
- 决策清单（第 3 节）只保留编号 + 一句话；详情永远去 `决策记录.md` 查。
- 同一位置的累积修订 → **覆盖**而非追加（如 ADR 数量、CICD 项数）。
- `current_state.json` 用覆盖更新，不追加历史；历史永远进会话日志。
- 触发瘦身时本身**不算重大变更**，无需新加 ADR；仅在会话日志记一笔。
**不触发条件**（避免噪声）：
- 单纯的代码风格修复 / typo
- 不涉及决策的纯实现工作（在会话日志记一笔即可）

**示例格式**：`| 2026-06-13 | 一句话摘要 | 链接 |`；旧条目按周 / 月聚合即可。
