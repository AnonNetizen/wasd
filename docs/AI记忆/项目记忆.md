## 1. 项目快照

- **名称**：wasd
- **类型**：俯视角 Roguelike 弹幕生存（《以撒的结合》+《吸血鬼幸存者》风格）
- **引擎**：Godot 4.6.3 + GDScript（强制类型化）
- **当前状态**：**文档/框架设计阶段**，无代码
- **设计文档版本**：v1.5
- **仓库结构**：`docs/`（文档）、`client/`（完整项目 Godot 客户端，占位）、`server/`（暂占位）、`MinimumViableProduct/`（MVP 隔离实验区）、`draft/` / `DRAFT/`（AI 禁区）
- **远端**：`https://github.com/AnonNetizen/wasd.git`
- **License**：MIT

## 2. 核心设计支柱（一眼回忆）

| 支柱 | 一句话 |
|------|-------|
| 数据驱动 | 所有可调数值进 `client/data/*.json`，零代码调参 + 热重载 |
| 遗物 = 数据 | `modifiers`（数值）+ `behaviors`（行为）描述，**严禁为每个遗物写硬编码分支** |
| 扩展优先 | 默认玩法规则不是硬编码上限；破限角色 / 道具必须用 capability、tag、primitive 或可复用 strategy 声明式扩展 |
| **完整 autoload 矩阵** | 3 协作（`Localization`/`Settings`/`Analytics`）+ 2 确定性（`RNG`/`GameClock`）+ 1 回放（`Replay`）+ 3 协调（`GameState`/`UIManager`/`PoolManager`）+ 2 资源（`SaveManager`/`AudioManager`）+ AI 协作工程（`docs/AI协作/`） |
| 玩家恒居中 | 相机挂玩家下，**关 limit 与 drag margin** |
| 词表契约 | 所有约定字符串走 `docs/词表与契约.md` 白名单，**禁止裸字符串/编造 id** |
| 代码-文档同步 | 长期代码模块 / 公共 API / 数据 schema / 依赖变化必须同步 `docs/代码/` 模块文档 |
| 默认中文沟通 | AI 面向用户的回复 / 计划 / 总结默认中文，引用代码、API、日志等原文时保留原语言 |
| AI 友好工程 | 项目索引、词表、数据校验、黄金样例、ADR、模板齐备；**任务模板 + 上下文预算 + 角色分工 + 实时验证回路** |
| 暂停 | `get_tree().paused` + 菜单节点 `process_mode=ALWAYS`，键走 InputMap action `pause` |
| 确定性 | 随机走 `RNG`、时间走 `GameClock`，禁止裸 `randi()` / `Time.get_ticks_msec()`（回放回归的硬约束） |

## 3. 既定决策（ADR 摘要，详见 `docs/决策记录.md`）

38 条已落地决策（详见 `docs/决策记录.md`）。摘要：

**早期（1~12）**：玩法定位 / 引擎 / 摄像机 / 输入 / 数据驱动遗物 / 数值外置 / 框架级 i18n / Settings / Analytics / E~I / AI 友好工程 / 元规则。

**框架夯实（13~19）**：暂停 / 三段仓库 / AI 记忆体系 / 回放（4th 基础设施） / 平衡接口 / AI 协作工程（5th） / 记忆自动瘦身。

**结构性增强（20~28，J~R 全采纳）**：
- 20 J. RNG 子流（spawn/drop/combat/ui_choice/world/meta）
- 21 K. GameState 流程统一（PauseMenu/Replay/Analytics 收口）
- 22 L. PoolManager 通用对象池（池 id 词表 §8）
- 23 M. UIManager 界面栈（modal/pauses_game/music_duck 元数据）
- 24 N. 伤害管线 + StatusEffect（DamageInfo / 五种 stack_rule）
- 25 O. SaveManager（version + 迁移链 + 损坏备份）
- 26 P. GameClock（暂停冻结 / 慢动作 / 假时钟注入）
- 27 Q. AudioManager（统一播放 + Bus 路由 + ducking）
- 28 R. 词表→代码常量（脚本生成 `client/scripts/contracts/*.gd`）
- 29 测试金字塔（`docs/测试策略.md`，规则 14-C）：L0~L5 五层 + 里程碑矩阵 + 性能预算 + 手动 checklist + AI 测试义务表
- 30 codebuddy 3 subagents + 5 slash commands（角色分工落地为可执行实体）
- 31 `.codebuddy/` / `.codex/` 多平台适配规则（规则 21）：核心规则语义一致，但允许按不同 AI agent / 平台优化配置，不要求字节级镜像
- 32 通用 AI 入口 + Codex/OpenCode 轻量入口：`AGENTS.md` 为通用入口，`CODEX.md` / `OPENCODE.md` 只做平台加载适配
- 33 `.opencode/` OpenCode 平台配置：`opencode.json` + 3 agents + 5 commands + rules，核心语义与 `.codebuddy/` / `.codex/` 同步
- 34 扩展优先 / 破限内容架构：角色、遗物、道具破默认限制时走 capability / tag / primitive / strategy，不写 id 特殊分支
- 35 代码-文档同步规范：长期代码模块 / 公共 API / 数据 schema / 依赖变化同步 `docs/代码/` 模块文档，避免文档漂移但不逐文件强制成文
- 36 AI 默认中文沟通：面向用户默认中文，用户要求或引用代码 / API / 日志原文等特殊场景例外
- 37 `draft/` / `DRAFT/` 人工草稿禁区：AI 默认不得读取 / 搜索 / 修改 / 整理 / 引用，除非用户明确点名授权；无需逐次汇报
- 38 `MinimumViableProduct/` MVP 隔离实验区：MVP 文档和独立客户端放该目录；初始规则为玩家不移动、四方向射击、四方向刷怪

## 9. 更新约定（重要）

**触发条件（满足任一即更新）**：
- 任何 ADR 新增 / 修改
- 任何编码规则新增 / 修改
- 任何"待决策项"被采纳或新增
- 任何重要文件结构调整
- 任何工具链变更（CI、git 配置、新模板）
- 重要里程碑达成

**更新动作**（AI 主动执行，无需用户提醒）：
1. 修订本文件第 1~5 节（项目快照 / 决策 / 待决策 / 工具链）以反映现状
2. 在第 6 节追加当日条目（**一行表格条目 + 日志链接**，不展开细节）
3. 在 `会话日志/YYYY-MM-DD.md` 写当日详细摘要（输入/产出/决策）
4. 修订第 7 节"下一步候选"

**节制原则**：
- 只记**结论 + 关键参数 + 文件指针**，不复述完整对话
- 单次会话总更新量目标 < 1KB
- 历史细节归入会话日志，主索引保持精简（< 200 行最佳）

**自动瘦身（无需用户提醒）**：
- 每次更新本文件前先检查总行数；**临近 200 行立即瘦身**——不等用户问。
- 第 6 节单日条目超过 1 行 → 折叠为「日期 + 一句话摘要 + 日志链接」格式。
- 7 天前的条目 → 压缩为「YYYY-MM-W（周）：一句话」周聚合。
- 30 天前的条目 → 整体折叠为「月份 + 该月主要里程碑」月聚合。
- 决策清单（第 3 节）只保留编号 + 一句话；详情永远去 `决策记录.md` 查。
- 同一位置的累积修订 → **覆盖**而非追加（如 ADR 数量、CICD 项数）。
- 触发瘦身时本身**不算重大变更**，无需新加 ADR；仅在会话日志记一笔。

**不触发条件**（避免噪声）：
- 单纯的代码风格修复 / typo
- 不涉及决策的纯实现工作（在会话日志记一笔即可）

**示例格式**：`| 2026-06-13 | 一句话摘要 | 链接 |`；旧条目按周 / 月聚合即可。
