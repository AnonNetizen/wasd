## 2. 核心设计支柱（一眼回忆）

| 支柱 | 一句话 |
|------|-------|
| 数据驱动 | 所有可调数值进 `client/data/`，平表数值优先 CSV、复杂配置优先 JSON；字段说明进 `client/data/README.md`，零代码调参 + 热重载 |
| 双语本地化 | 玩家可见文本进 `client/locale/strings.csv`，当前首批维护 `zh_CN` / `en`；AI 自动补另一语言首版译文，人工最终复核 |
| 深局外成长 | `meta_progression.json` 数据化局外货币、账号等级、永久升级和解锁，状态走 `SaveManager` 的 `meta` kind |
| 强玩家存档 | `SaveManager` 同时支持 `meta` 局外成长和 `run` 暂停退出续局，要求迁移、原子写入、备份回退和损坏隔离 |
| 遗物 = 数据 | `modifiers`（数值）+ `behaviors`（行为）描述，**严禁为每个遗物写硬编码分支** |
| 扩展优先 | 默认玩法规则不是硬编码上限；破限角色 / 道具必须用 capability、tag、primitive 或可复用 strategy 声明式扩展 |
| 模式友好资源复用 | 角色、遗物、道具、敌人、成长奖励等资源本体默认模式无关；游戏模式只通过资源池、权重、条件、禁用列表和轻量覆盖组合资源 |
| 未来多人友好边界 | 当前只做单人，不实现网络；输入、伤害、阵营、回放、存档和埋点预留 actor / participant / team / intent 概念 |
| **完整 autoload 矩阵** | 3 协作（`Localization`/`Settings`/`Analytics`）+ 2 确定性（`RNG`/`GameClock`）+ 1 回放（`Replay`）+ 3 协调（`GameState`/`UIManager`/`PoolManager`）+ 2 资源（`SaveManager`/`AudioManager`）+ AI 协作工程（`docs/AI协作/`） |
| 玩家恒居中 | 相机挂玩家下，**关 limit 与 drag margin** |
| 词表契约 | 所有约定字符串走 `docs/词表与契约.md` 白名单，**禁止裸字符串/编造 id** |
| 代码-文档同步 | 长期代码模块 / 公共 API / 数据 schema / 依赖变化必须同步详细的 `docs/代码/` 模块文档 |
| AI 知识库 | `docs/AI知识库索引.md` + `docs/_kb_index.json` + `docs/术语表.md` + `tools/docs_health_check.py` |
| 可验证数据闭环 | `tools/sync_contracts.py` 生成 `_contracts.json` 与 `client/scripts/contracts/*.gd`；`tools/validate_data.py` 校验正式数据 / locale；`tools/test_data_loader_schema.py` 固定坏数据回归；`tools/lint_gdscript_rules.py` 固定第一档 GDScript 项目 lint；`tools/lint_project_rules.py` 固定第二档项目规则 lint；`tools/lint_semantic_rules.py` 固定第三档非阻塞语义 advisory；CI 已接入 |
| 轻量 Godot Bridge | `tools/godot_bridge.py` 默认操作正式 `client/`，支持 `export-tree`、`validate-data`、`godot-version`、`headless-boot`、`runtime-smoke`、`settings-smoke`、`meta-smoke`、`save-smoke`、F8 `l1-smoke`、`replay-smoke`、`replay-runner`、`replay-runner --rerun-runtime-summary`、`replay-input-smoke`、`capture-golden-replay`、`capture-golden-replay --golden-scenario golden_pause_resume`、`capture-golden-replay --golden-scenario golden_full_death`、`capture-golden-replay --golden-scenario golden_level_up_choice`、`perf-probe` |
| 默认中文沟通 | AI 面向用户的回复 / 计划 / 总结默认中文，引用代码、API、日志等原文时保留原语言 |
| 事实审查 / 需求评估 | 用户问有没有问题 / 风险时不硬找问题；用户提需求后先反馈价值、性价比、复杂度和主要风险 |
| AI 友好工程 | 项目索引、词表、数据 / 文案配置手册、数据校验、黄金样例、ADR、模板齐备；**任务模板 + 上下文预算 + 角色分工 + 实时验证回路 + 工具先行代码审核流程** |
| 低 token 开工 | 日常接手先读 `docs/AI协作/快速开工.md` + `current_state.json`，完整长期记忆 / GDD / ADR / 测试策略按任务触发读取 |
| 阶段工作包 | 正式项目阶段任务优先读 `docs/AI协作/工作包/`，代码阶段优先模块文档 + 目标源码 |
| 暂停 | `get_tree().paused` + 菜单节点 `process_mode=ALWAYS`，键走 InputMap action `pause` |
| 确定性 | 随机走 `RNG`、时间走 `GameClock`，禁止裸 `randi()` / `Time.get_ticks_msec()`（回放回归的硬约束） |

## 9. 更新约定（重要）

**触发条件（满足任一即更新）**：
- 任何 ADR 新增 / 修改
- 任何编码规则新增 / 修改
- 任何"待决策项"被采纳或新增
- 任何重要文件结构调整
- 任何工具链变更（CI、git 配置、新模板）
- 重要里程碑达成

**更新动作**（AI 主动执行，无需用户提醒）：
1. 修订本文件第 1~5 节（项目快照 / 决策 / 待决策 / 工具链）以反映长期事实
2. 更新 `docs/AI记忆/current_state.json`（当前阶段、最新 ADR、待决策项、下一步、最近验证）
3. 在第 6 节保留 / 覆盖当日一行摘要（**一行表格条目 + 日志链接**，不展开细节）
4. 在 `会话日志/YYYY-MM-DD.md` 写当日详细摘要（输入/产出/决策/验证）

**节制原则**：
- 只记**结论 + 关键参数 + 文件指针**，不复述完整对话
- 单次会话总更新量目标 < 1KB
- 历史细节归入会话日志，短期状态归入 `current_state.json`，长期索引保持精简（< 200 行最佳）

**自动瘦身（无需用户提醒）**：
- 每次更新本文件前先检查总行数；**临近 200 行立即瘦身**——不等用户问。
- 第 6 节单日条目超过 1 行 → 折叠为「日期 + 一句话摘要 + 日志链接」格式。
- 7 天前的条目 → 压缩为「YYYY-MM-W（周）：一句话」周聚合。
- 30 天前的条目 → 整体折叠为「月份 + 该月主要里程碑」月聚合。
- 决策清单（第 3 节）只保留编号 + 一句话；详情永远去 `决策记录.md` 查。
- 同一位置的累积修订 → **覆盖**而非追加（如 ADR 数量、CICD 项数）。
- `current_state.json` 用覆盖更新，不追加历史；历史永远进会话日志。
- 触发瘦身时本身**不算重大变更**，无需新加 ADR；仅在会话日志记一笔。
**不触发条件**（避免噪声）：
- 单纯的代码风格修复 / typo
- 不涉及决策的纯实现工作（在会话日志记一笔即可）

**示例格式**：`| 2026-06-13 | 一句话摘要 | 链接 |`；旧条目按周 / 月聚合即可。
