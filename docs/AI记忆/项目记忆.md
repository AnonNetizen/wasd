## 1. 项目快照

- **名称**：wasd
- **作者署名**：中文 `伦敦阿农`；英文 `Anon London`
- **类型**：俯视角射击刷宝生存（手动按住开火的射击身份 + 《星际战甲》/《暗黑》刷装备与词条长期追求 + 手工房间串联短刷图 + 以撒式道具 / 机关 / 构筑组合；玩法判定与显示都保持 2D 平面）
- **IP 方向**：ADR #88 已定为 **《破巢者》**（英文暂定 `Nestbreakers`），独立权威文档为 `docs/IP设定.md`：未知原因导致其他宇宙与本宇宙的通道突然打开，原本拥有银河系星际文明的人类被打散，只剩首都星域相对稳定；多英雄主动突入敌方“巢”，在怪潮中夺取遗物、升级构筑并尝试打穿巢核、切断通道或削弱敌方源头；“巢”是敌方核心据点 / 生产源头 / 通道锚点 / 意志中枢的泛称，不限定为虫巢。ADR #111 新增 `docs/IP美术风格.md` 作为 IP 美术权威：首个敌巢视觉方向是“被异宇宙规则钙化出来的活体建筑”，敌巢 / 虫族占用青 / 红 / 白和骨白 / 蜡黄 / 干肉粉 / 黑紫，玩家与玩家子弹默认避开青 / 红 / 白，敌方远程攻击可用红色，宝箱和地图兴趣点按功能色区分。
- **引擎**：Godot 4.7.1 stable + GDScript（强制类型化；新写 / 修改脚本以 Godot 4.7 官方 GDScript style guide 为基础风格）

## 9. 更新约定（重要）

**触发条件（满足任一即更新）**：
- 任何 ADR 新增 / 修改
- 任何编码规则新增 / 修改
- 任何"待决策项"被采纳或新增
- 任何重要文件结构调整
- 任何工具链变更（CI、git 配置、新模板）
- 重要里程碑达成

**更新动作**（AI 主动执行，无需用户提醒）：
1. 修订本文件第 1~5 节（项目快照 / 决策 / 待决策 / 工具链）以反映长期事实
2. 更新 `docs/AI记忆/current_state.json`（当前阶段、最新 ADR、待决策项、下一步、最近验证）
3. 在第 6 节保留 / 覆盖当日一行摘要（**一行表格条目 + 日志链接**，不展开细节）
4. 在 `会话日志/YYYY-MM-DD.md` 写当日详细摘要（输入/产出/决策/验证）

**节制原则**：
- 只记**结论 + 关键参数 + 文件指针**，不复述完整对话
- 单次会话总更新量目标 < 1KB
- 历史细节归入会话日志，短期状态归入 `current_state.json`，长期索引保持精简（< 200 行最佳）

**自动瘦身（无需用户提醒）**：
- 每次更新本文件前先检查总行数；**临近 200 行立即瘦身**——不等用户问。
- 第 6 节单日条目超过 1 行 → 折叠为「日期 + 一句话摘要 + 日志链接」格式。
- 7 天前的条目 → 压缩为「YYYY-MM-W（周）：一句话」周聚合。
- 30 天前的条目 → 整体折叠为「月份 + 该月主要里程碑」月聚合。
- 决策清单（第 3 节）只保留编号 + 一句话；详情永远去 `决策记录.md` 查。
- 同一位置的累积修订 → **覆盖**而非追加（如 ADR 数量、CICD 项数）。
- `current_state.json` 用覆盖更新，不追加历史；历史永远进会话日志。
- 触发瘦身时本身**不算重大变更**，无需新加 ADR；仅在会话日志记一笔。
**不触发条件**（避免噪声）：
- 单纯的代码风格修复 / typo
- 不涉及决策的纯实现工作（在会话日志记一笔即可）

**示例格式**：`| 2026-06-13 | 一句话摘要 | 链接 |`；旧条目按周 / 月聚合即可。

> 维护要求：本文件是 AI 协作的"长期记忆主控"。任何与本约定不一致的更新，应在 `docs/决策记录.md` 中以新 ADR 改写约定，再同步修订本文件。
