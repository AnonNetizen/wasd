# IP 美术风格

> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。
> 本文档是《破巢者》IP 美术风格、敌巢视觉语言、阵营色与地图兴趣点功能色的权威入口；改美术主口径、阵营色、兴趣点色彩或资产 brief 规则时，必须同步 `docs/IP设定.md`、`docs/游戏设计文档.md`、`docs/AI导航.md`、`docs/术语表.md`、`docs/决策记录.md`、`docs/AI记忆/项目记忆.md` 与 `docs/AI记忆/current_state.json`。

## 1. 核心美术句

《破巢者》的首个敌巢视觉方向是：**被异宇宙规则钙化出来的活体建筑**。

它有虫、有肉、有壳、有巢核和外骨骼，但不是传统湿漉漉的虫巢。它更像某种外宇宙生物工业：骨质地面、蜂蜡般的壳层、干肉色膜片、深红裂缝、黑紫深处阴影，以及少量青色巢核能量。

目标读法：

- 第一眼能读出“虫巢 / 活体结构”。
- 第二眼能读出“这不是自然生态，而是外宇宙规则改写出的生物工业”。
- 实际战斗中，玩家、玩家弹幕、敌人、敌方远程攻击、宝箱和地图兴趣点要能用颜色快速分辨。

## 2. 参考边界

可参考的方向：

- 钙化骨壳、蜂蜡质感、外骨骼拱肋、虫巢孔洞。
- 干燥肉膜、深红裂缝、黑紫深腔，而不是大面积湿肉、黏液或血浆。
- 俯视角 / 像素概念图的玩法可读性：中场地面清楚，边缘建筑复杂。
- 巢核、精英弱点和异宇宙机器感用少量高亮点缀，而不是全场发光。

需要避免：

- 只像普通虫族或普通异形巢穴。
- 全场同色，导致玩家、敌人、子弹和交互物混在一起。
- 毒蓝 / 青色过量，失去“重要交互 / 巢核 / 弱点”的信号价值。
- 所有资产都矩形化或菱形化。当前地图、危险地块、贴地范围和 AOE 默认用水平 / 垂直矩形俯视格，非地面资产仍按清晰俯视轮廓落地。

## 3. 敌巢主色板

敌巢、虫族、巢核结构和敌方生物工业使用以下主色：

| 色名 | Hex | 主要用途 | 读图含义 |
|------|-----|----------|----------|
| 骨白 | `#D8C7A1` | 地面骨板、外骨骼、壳壁、虫族甲壳高光 | 钙化、外骨骼、可行走表面 |
| 蜡黄 | `#B88B4A` | 蜂蜡状壳层、老化骨板、边缘墙体 | 生物工业、虫巢材料 |
| 干肉粉 | `#A85F5F` | 肉膜、软组织、虫族关节 | 活体结构但不湿滑 |
| 深红 | `#5A1620` | 裂缝、危险膜、敌方远程攻击、敌方伤害提示 | 敌意、危险、破裂组织 |
| 黑紫 | `#1B1024` | 深处阴影、洞口、外圈虚空、不可达区域 | 深度、未知、边界 |
| 毒蓝 / 冷青 | `#3FE7D4` | 巢核、精英弱点、虫族眼点、巢穴终端、少量交互节点 | 异宇宙能量、敌巢系统、关键弱点 |

用法比例建议：

- 地面 / 巢壁：骨白 + 蜡黄为主，约占敌巢可见面积的大部分。
- 裂缝 / 肉膜：干肉粉 + 深红作为结构切口和危险暗示，不铺满中场。
- 深处阴影：黑紫用于边缘、洞口、墙后和不可达深层。
- 高亮点：毒蓝只给巢核、精英弱点、巢穴终端、特殊敌巢交互，必须克制。

## 4. 阵营色规则

为了避免敌我混色，当前首批视觉规则如下：

| 归属 | 可用色 | 禁用 / 避免 | 说明 |
|------|--------|-------------|------|
| 敌巢环境 | 骨白、蜡黄、干肉粉、深红、黑紫、毒蓝 | 大面积纯绿 / 金 / 紫 | 环境是“敌方宇宙规则”的底色 |
| 虫族敌人 | 骨白、蜡黄、干肉粉、深红、黑紫、毒蓝弱点 | 玩家阵营金紫绿主色 | 敌人可使用青、红、白，让它们归属于敌巢 |
| 敌方远程攻击 | 深红 / 红色 | 玩家弹幕色 | 红色优先表达敌方远程威胁和危险提示 |
| 玩家角色 | 紫、金、琥珀、橄榄绿、黑金属 | 青、红、白 | 玩家不使用虫族占用的青、红、白，避免阵营混淆 |
| 玩家子弹 | 金、琥珀、紫、橄榄绿 / 酸绿 | 青、红、白 | 玩家默认弹幕不得使用青、红、白；青留给巢核 / 虫族弱点，红留给敌方攻击 |
| 中立 / 奖励 | 金、绿、紫、蓝紫、橙 | 与敌方攻击红混淆 | 宝箱、遗物、治疗、升级站等用功能色区分 |

原则：**青、红、白给虫族 / 敌巢用；玩家和玩家子弹不用青、红、白；敌人的远程攻击可以用红色。**

## 5. 地图兴趣点功能色

地图兴趣点要比环境更“像游戏物件”。它们可以造型上带骨壳、蜡质或活体结构，但功能色必须独立、稳定。

| 兴趣点 | 推荐颜色 | 视觉包装 | 读图目标 |
|--------|----------|----------|----------|
| 宝箱 / 战利箱 | 金 + 祖母绿 | 外骨壳箱体、宝石锁芯、机械肋骨扣件 | “奖励 / 值得靠近” |
| 遗物祭坛 | 紫 / 蓝紫 | 骨质底座、悬浮遗物、异宇宙铭纹 | “构筑选择 / 稀有内容” |
| 治疗点 | 绿色 / 橄榄绿 | 生命胶囊、修复孢囊、医疗补给舱 | “回复 / 安全收益” |
| 升级 / 改造站 | 橙 / 琥珀 | 生物工业接口、开壳装置、强化台 | “强化 / 消耗资源” |
| 武器缓存 | 琥珀 / 金属黑 | 星舰补给箱、武器架、加固锁 | “武器 / 弹药 / 模组” |
| 巢穴终端 | 毒蓝 / 冷青 | 巢核接口、界门控制点、虫族能量柱 | “敌巢系统 / 高价值交互” |
| 危险门 / 封印裂隙 | 深红 | 红色膜门、裂缝、敌方符号 | “危险 / 需要准备” |

不要把所有兴趣点都做成同一种青色发光物。青色在本项目中更接近敌巢系统色，不能泛化成“任何可互动都发青光”。

## 6. 子弹与特效读图

玩家弹幕和敌方弹幕必须分开：

- 玩家默认射击：金 / 琥珀，适合持续弹流。
- 玩家稀有技能或高阶构筑：紫 / 蓝紫，适合遗物或超载感。
- 玩家特殊毒 / 酸 / 生化效果：橄榄绿或酸绿，但避免接近毒蓝。
- 敌人远程攻击：红色，允许强警示。
- 敌巢弱点 / 巢核能量：毒蓝 / 冷青，不等同于玩家攻击色。

如果某个玩家技能确实需要“敌巢能量反用”的叙事，可以在文案和 VFX brief 中明确它是夺取敌巢力量后的例外；默认仍不要用青、红、白做玩家主弹幕。

## 7. 俯视角落地

本文件只规定 IP 与颜色语言；具体资产仍必须遵守 GDD §8.2-A 的俯视资产落地规则：

- 贴地范围、机关、AOE、地面符号优先使用矩形 / 方形俯视格或清晰贴地轮廓，并与当前量化矩形地图格对齐。
- 角色、敌人、宝箱、拾取物、子弹、障碍物和特效不强制矩形。
- 非地面资产必须声明 `footprint`、`anchor_point`、`orientation_read`、`sort_layer` 和真实碰撞 / 触发形状。
- 中场战斗区域尽量保持骨白 / 蜡黄低饱和大块面，复杂肉膜、孔洞和高亮尽量放到边缘或兴趣点上，避免战斗读图变脏。

## 8. 资产 Brief 模板

生成或替换非 UI 资产时，至少补齐以下字段：

```text
asset_type:
faction_or_function:
palette_role:
allowed_colors:
forbidden_colors:
footprint_shape:
anchor_point:
shadow:
sort_layer:
collision_or_trigger_shape:
readability_note:
```

示例：

```text
asset_type: projectile
faction_or_function: player_default_bullet
palette_role: player_combat
allowed_colors: gold, amber, violet
forbidden_colors: cyan, red, white
footprint_shape: point
anchor_point: projectile center
shadow: none
sort_layer: air
collision_or_trigger_shape: circle hit radius from data
readability_note: must separate clearly from red enemy projectiles and cyan nest weak points
```

## 9. 当前结论

当前推荐把这套文档作为首个正式 IP 美术基准：

- 敌巢和虫族占用青 / 红 / 白，以及骨白 / 蜡黄 / 干肉粉 / 黑紫。
- 玩家和玩家弹幕避开青 / 红 / 白，使用紫 / 金 / 琥珀 / 绿 / 黑金属。
- 敌方远程攻击用红色。
- 宝箱和地图兴趣点按功能色区分，不和虫族底色混在一起。
- 巢穴美术要像“外宇宙生物工业”，不是单纯湿肉虫巢。
