# 词表与契约（约定字符串白名单）

> 本文件是项目所有"约定字符串"的**唯一白名单**。代码中必须以**常量/枚举**引用这里的 id，**禁止裸字符串、禁止编造未登记的 id**。
> 新增 id 的流程：① 在本文件登记 → ② 在逻辑层实现对应原语/处理 → ③ 才能在数据/代码中使用。
> 配套：`AI导航.md`、`游戏设计文档.md`、当前平台编码规则入口（见 `AGENTS.md` 第 3 步）。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是约定字符串权威；新增 / 删除 / 改名 id 时，常见联动为 `tools/sync_contracts.py` 生成产物（落地后）、引用该 id 的数据 / 代码、`docs/AI导航.md`、`docs/决策记录.md`、`docs/AI记忆/项目记忆.md`。

## 1. 属性 stat（用于 modifiers）
最终属性 = `(基础值 + Σ加法修正) × Π乘法修正`。

| stat id | 含义 | 单位/类型 |
|---------|------|-----------|
| `max_hp` | 最大生命 | float |
| `health_regen` | 每秒自动生命恢复 | HP/s，float |
| `move_speed` | 移动速度 | px/s |
| `damage_invulnerability_duration` | 受伤后无敌时间 | 秒，float |
| `player_separation_radius` | 玩家中心排斥半径 | px |
| `damage` | 子弹伤害 | float |
| `fire_rate` | 射速（每秒发数） | float |
| `bullet_speed` | 子弹飞行速度 | px/s |
| `bullet_range` | 子弹最大射程 | px |
| `bullet_count` | 同时发射数 | int |
| `pierce_count` | 可额外命中的伤害目标数量；不影响墙体 | int |
| `wall_pierce` | 是否忽略全部地形；`0` 阻挡，`>0` 穿墙 | float |
| `crit_chance` | 暴击率 | 0~1 |
| `crit_mult` | 暴击倍率 | float |
| `pickup_range` | 拾取范围 | px |
| `pickup_orb_speed` | 经验球吸附速度 | px/s |
| `luck` | 幸运（影响掉落、稀有度、升级候选是否 4 选 1） | float |
| `armor` | 物理防御 | float |
| `resist_fire` / `resist_poison` / `resist_lightning` | 元素抗性 | 0~1 |
| `lifesteal_ratio` | 吸血比例 | 0~1 |

> 修正器格式：`{ "stat": "<stat id>", "type": "add" | "mult", "value": <number> }`

## 6. 本地化 key 命名规范（见设计文档 9.4）
- 文案配置与人工翻译流程见 `client/locale/README.md`；所有玩家可见译文集中写入 `client/locale/strings.csv`。
- 当前必填语言为 `zh_CN` 与 `en`；新增文案时 AI 必须自动补齐另一语言首版译文，人工最终复核。
- 统一蛇形小写：`<域>_<对象>_<字段>`。
- 域前缀约定：
  - `ui_`：界面文本（`ui_settings` / `ui_resume` / `ui_pause` / `ui_quit`）；装备 Mod 面板使用 `ui_gear_mod_` 子前缀
  - `character_`：角色（`character_default_name` / `character_default_desc`）
  - `weapon_`：武器（`weapon_basic_blaster_name` / `weapon_basic_blaster_desc`）
  - `relic_`：遗物（`relic_sharp_rounds_name` / `relic_sharp_rounds_desc`）
  - `item_`：道具
  - `skill_`：技能（名称、描述、提示）
  - `gear_mod_`：装备 Mod（名称、描述、升级资源）
  - `enemy_`：敌人名
  - `hazard_`：机关 / 危险物名
  - `hint_`：提示/教程
- 名称用 `_name`，描述用 `_desc`；动态数值用占位符 `{value}`，禁止字符串拼接。

## 7. 输入动作 action（InputService / GUIDE，可在设置重绑定）
业务代码只以自动生成的 `Actions` 常量引用下列 id，并通过 `InputService` 消费归一化 intent；**禁止硬编码物理按键、手柄按钮或手柄轴，也禁止业务直接读取 GUIDE / `Input` / `InputMap`**。GUIDE 是物理映射引擎，`InputService` 是项目唯一业务边界；`InputMap` 只允许用于 GUIDE 内部、`InputService` 的 Godot `Control` UI 兼容桥和测试边界。

| action id | 含义 | 默认输入 |
|-----------|------|----------|
| `move` | 归一化 `Vector2` 移动 intent | W / S / A / D；手柄左摇杆 |
| `aim` | 归一化 `Vector2` 瞄准 intent；玩家适配层在鼠标位置和方向输入之间选择最近有效来源 | 鼠标位置；↑ / ↓ / ← / →；手柄右摇杆 / D-pad |
| `pointer_position` | 内部 `Vector2` viewport pointer action，只供 `InputService` 的鼠标瞄准适配 | 鼠标位置；不可在设置解绑 |
| `fire` | 主武器开火；按住持续射击，松开停火 | 鼠标左键；手柄右扳机 |
| `use_active_item` | 释放主动道具 | Space；手柄 A / South |
| `interact` | 打开宝箱 / 交互物 | E；手柄 X / West |
| `show_stats_panel` | 按住显示详细数值面板；松开隐藏，期间不暂停 | Tab |
| `pause` | 暂停 / 继续（切换） | Esc；手柄 Start / Menu |
| `ui_up` / `ui_down` / `ui_left` / `ui_right` | UI 焦点导航；由 `InputService` 窄桥接到 Godot 内置 `ui_*` | 方向键 / WASD；手柄左摇杆 / D-pad |
| `ui_confirm` | UI 确认 / 激活当前焦点 | Enter；手柄 A / South |
| `ui_back` | UI 返回 / 关闭栈顶面板 | Esc；手柄 B / East |
| `move_up` / `move_down` / `move_left` / `move_right` | deprecated：仅用于 replay/settings v1 内存迁移，正式业务不再读取 | 旧版 W / S / A / D |
| `aim_up` / `aim_down` / `aim_left` / `aim_right` | deprecated：仅用于 replay/settings v1 内存迁移，正式业务不再读取 | 旧版方向键 / 右摇杆 / D-pad |
| `debug_toggle_console` | 调试：切换 GM 指令控制台（开发期） | F1 / ` |
| `debug_submit_command` | 调试：提交 GM 指令（开发期） | Enter |
| `debug_close_console` | 调试：关闭 GM 指令控制台（开发期） | Esc |
| `debug_toggle_replay` | 调试：切换回放视图（开发期） | F9 |

> 暂停统一走 `GameState`（GDD 9.12）；暂停菜单等需暂停时仍响应的节点设 `process_mode = PROCESS_MODE_ALWAYS`。
> 调试用 action 仅在 debug build 启用，发布版禁用。

### 7.1 重绑定 binding id

设置页与 `InputService` 以稳定 binding id 标识“一个玩家可见的绑定槽”，binding id 不等于 GUIDE action id。方向键子绑定与摇杆组可以共同驱动同一个 `Vector2` action；业务代码不得读取 binding id。旧 `settings.cfg` v1 曾使用同名 `input.*` key 的条目只在首次迁移时读取，当前绑定权威统一是 `user://input_bindings.tres`。

| binding id | 对应能力 | 当前槽位边界 |
|------------|----------|--------------|
| `input.move_up` / `input.move_down` / `input.move_left` / `input.move_right` | `move` 的键鼠四方向子绑定 | 每方向一个键鼠槽 |
| `input.move_stick` | `move` 的手柄二维轴组 | 一个手柄摇杆组，默认左摇杆 |
| `input.aim_up` / `input.aim_down` / `input.aim_left` / `input.aim_right` | `aim` 的键鼠四方向子绑定 | 每方向一个键鼠槽；D-pad 兜底固定 |
| `input.aim_stick` | `aim` 的手柄二维轴组 | 一个手柄摇杆组，默认右摇杆 |
| `input.fire` | `fire` | 一组键鼠槽和一组手柄槽 |
| `input.use_active_item` / `input.interact` / `input.show_stats_panel` | 对应布尔 gameplay action | 各一组键鼠槽和一组手柄槽 |
| `input.pause` / `input.ui_confirm` / `input.ui_back` | 暂停与 UI 布尔 action | 各一组键鼠槽和一组手柄槽；退出恢复键另有不可重绑定兜底 |
