# 词表与契约（约定字符串白名单）

> 本文件是项目所有"约定字符串"的**唯一白名单**。代码中必须以**常量/枚举**引用这里的 id，**禁止裸字符串、禁止编造未登记的 id**。
> 新增 id 的流程：① 在本文件登记 → ② 在逻辑层实现对应原语/处理 → ③ 才能在数据/代码中使用。
> 配套：`AI导航.md`、`游戏设计文档.md`、当前平台编码规则入口（见 `AGENTS.md` 第 3 步）。
>
> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是约定字符串权威；新增 / 删除 / 改名 id 时，常见联动为 `tools/sync_contracts.py` 生成产物（落地后）、引用该 id 的数据 / 代码、`docs/AI导航.md`、`docs/决策记录.md`、`docs/AI记忆/项目记忆.md`。

## 1. 属性 stat（用于 modifiers）
最终属性 = `(基础值 + Σ加法修正) × Π乘法修正`。

| stat id | 含义 | 单位/类型 |
|---------|------|-----------|
| `max_hp` | 最大生命 | int |
| `move_speed` | 移动速度 | px/s |
| `damage` | 子弹伤害 | float |
| `fire_rate` | 射速（每秒发数） | float |
| `bullet_speed` | 子弹飞行速度 | px/s |
| `bullet_count` | 同时发射数 | int |
| `pierce_count` | 穿透次数 | int |
| `crit_chance` | 暴击率 | 0~1 |
| `crit_mult` | 暴击倍率 | float |
| `pickup_range` | 拾取范围 | px |
| `luck` | 幸运（影响掉落/三选一） | float |
| `armor` | 物理防御 | float |
| `resist_fire` / `resist_poison` / `resist_lightning` | 元素抗性 | 0~1 |
| `lifesteal_ratio` | 吸血比例 | 0~1 |

> 修正器格式：`{ "stat": "<stat id>", "type": "add" | "mult", "value": <number> }`

## 6. 本地化 key 命名规范（见设计文档 9.4）
- 统一蛇形小写：`<域>_<对象>_<字段>`。
- 域前缀约定：
  - `ui_`：界面文本（`ui_settings` / `ui_resume` / `ui_pause` / `ui_quit`）
  - `relic_`：遗物（`relic_sharp_rounds_name` / `relic_sharp_rounds_desc`）
  - `item_`：道具
  - `enemy_`：敌人名
  - `hint_`：提示/教程
- 名称用 `_name`，描述用 `_desc`；动态数值用占位符 `{value}`，禁止字符串拼接。

## 7. 输入动作 action（InputMap，可在设置重绑定）
代码中只引用这些 action id（`Input.is_action_pressed("<id>")` 等），**禁止硬编码物理按键**；重绑定通过 `Settings` 写入 InputMap。

| action id | 含义 | 默认键 |
|-----------|------|--------|
| `move_up` / `move_down` / `move_left` / `move_right` | 移动（8 方向由组合得出） | W / S / A / D |
| `aim_up` / `aim_down` / `aim_left` / `aim_right` | 射击瞄准（4 方向） | ↑ / ↓ / ← / → |
| `use_active_item` | 释放主动道具 | Space |
| `pause` | 暂停 / 继续（切换） | Esc |
| `debug_toggle_replay` | 调试：切换回放视图（开发期） | F9 |

> 暂停统一走 `GameState`（GDD 9.12）；暂停菜单等需暂停时仍响应的节点设 `process_mode = PROCESS_MODE_ALWAYS`。
> 调试用 action 仅在 debug build 启用，发布版禁用。

### 12.2 能力 capability_id
代码引用走 `client/scripts/contracts/capabilities.gd`。Capability 表示“该内容被允许改写哪类默认规则”，具体效果仍需由 effect / behavior / strategy 实现。

| id | 允许突破的默认规则 |
|----|------------------|
| `cap_aim_mode_override` | 改写瞄准模型（如 4 方向 → 自动瞄准 / 鼠标瞄准） |
| `cap_fire_mode_override` | 改写开火模型（如自动开火 → 蓄力 / 手动 / 条件触发） |
| `cap_movement_model_override` | 改写移动模型（如冲刺、滑行、惯性、传送） |
| `cap_inventory_slot_override` | 改写栏位数量或携带限制（如额外主动栏） |
| `cap_camera_policy_override` | 改写默认摄像机策略（必须仍通过统一 Camera / GameState 管线） |

### 12.3 内容标签 content_tag
代码引用走 `client/scripts/contracts/content_tags.gd`。

| id | 含义 |
|----|------|
| `tag_character` | 角色内容 |
| `tag_relic` | 被动遗物 |
| `tag_active_item` | 主动道具 |
| `tag_consumable` | 消耗品 |
| `tag_limit_break` | 明确突破默认规则的内容，必须带 capability 与测试说明 |
