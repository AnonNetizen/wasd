# 简单设计思路（项目原点）

> **AI 修改说明**：修改本文档前先读 `docs/AI协作/文档维护指南.md`。本文档是项目原始需求记录，原则上保持历史原貌；若需求被正式采纳或改写，应同步 `docs/游戏设计文档.md`、`docs/决策记录.md`、`docs/AI记忆/项目记忆.md`，不要只改这里。

我希望设计一个rougelike游戏。核心机制类似于《以撒的结合》：
1.玩家可以通过wasd移动
2.玩家可以通过方向键控制射击方向，但不能控制是否射击。射击是全自动的。
3.取消《以撒的结合的》房间设计，改成类似于《吸血鬼幸存者》的大地图。
4.子弹有飞行速度，并不是立即到达目标。
5.有可充能主动道具和一次性消耗品，类似以撒的结合。
6.有被动遗物，类似以撒的结合。
7.有会让玩家受伤的机关、类似以撒的结合。
8.这款游戏需要在美术素材上简单，便于ai生成
9.游戏代码需要易于ai分析、理解、处理、生成
10.需要把要人处理的数值，比如人物、遗物的数值单独列出一个文件，方便人为调整。

正式采纳补充（2026-06-13）：完整项目需要支持手柄，默认映射为左摇杆移动、右摇杆 / D-pad 四方向瞄准、手柄 A / South 释放主动道具、Start / Menu 暂停；详情以 GDD、词表与 ADR #39 为准。
