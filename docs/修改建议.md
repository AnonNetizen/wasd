## 已采纳 A：射击改为按住开火（2026-06-24）
- 决策：ADR #109 已将默认武器改为按住 `fire` action（左键 / 右扳机）持续射击，松开停火；旧自动开火设置仅保留为兼容项，不作为当前玩法入口。
- 理由：手动控制开火能把核心操作从被动动作技能转向射击，给玩家保留不开火走位、调整站位和管理射击节奏的空间。

## 已采纳 B：默认鼠标瞄准（2026-06-20）
- 决策：键鼠默认改为鼠标相对玩家 / 视口中心方向瞄准，方向键 / 手柄右摇杆 / D-pad 作为兜底；玩家和敌人表现只做左 / 右两种朝向。
- 理由：开放地图中鼠标瞄准比默认 4 方向更直觉，能降低斜向敌人空打问题；左右朝向能保持占位表现成本可控。

## 已采纳 C：有限地图引导问题改由手工房间串联解决（2026-06-28）
- 决策：ADR #127 已把默认短刷图下一阶段转向手工房间串联；每个房间用 Godot `.tscn` + marker 手工制作，首片只做线性房间序列、清房开门和进入下一房间。
- 理由：相比继续在开放有限地图中补兴趣点，房间制能更直接给玩家短期目标和出入口节奏，也方便人工打磨敌人生成点、陷阱和门。

## D. 子弹飞行速度 + 按住开火的手感（原则 #4）
- 移动 + 按住开火在鼠标瞄准后更容易形成稳定枪线，但手柄 / 方向键兜底和高速敌人仍可能带来空打。
- **建议**：初期把 `bullet_speed` 调高、可加轻微「弹道宽容 / 小范围散射」，命中体验更好；后续用遗物（追踪弹）或自动瞄准模式补足。

## E. 升级选项池内容（成长系统）
- 已定：ADR #120 后默认标准模式暂不启用局内升级；未来显式挂接 `growth_pools` 的成长模式可默认 3 选 1，`luck` 可概率性把本次候选提升为 4 选 1。
- 待定：升级选项具体包含哪些类型，例如遗物、纯属性升级、主动道具强化、回血、刷新 / 跳过 / banish 等。
- **建议**：先用数据表定义选项池、稀有度权重、前置条件和排除规则，再由 `RNG.ui_choice` 结合 `luck` 抽取，避免把选项类型写死在升级系统中。
